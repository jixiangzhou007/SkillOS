## 一、愿景与成功定义

### 1.1 愿景

**让每一位员工在飞书 / Cursor 里用自然语言对话，把岗位经验沉淀为可复用、可验证、可共享的 Agent Skill**——不是又一份 AI 生成的 markdown，而是带「已验证 / 待确认」标签的公司知识资产。

### 1.2 全公司推广成功标准（12 个月）

| 维度 | 目标 |
|------|------|
| **覆盖** | ≥ 80% 业务部门各有 ≥ 3 个「已发布」技能 |
| **质量** | 已发布技能中 ≥ 70% 硬规则声明完成人工确认（verified） |
| **复用** | 技能被 Agent 调用 ≥ 500 次/月（全公司） |
| **效率** | 新人上手某流程时间缩短 ≥ 30%（抽样 3 个场景调研） |
| **治理** | 100% 技能有 owner、版本、审批记录 |
| **合规** | 通过信息安全评审；敏感数据零外泄事故 |

### 1.3 不在首期范围（刻意不做）

- 对外 SaaS 商业化 / 技能 marketplace 支付
- 52 格级自动进化 benchmark（Phase 7 研究项）
- 替换现有 ERP/CRM/审批系统

### 3.1 技能库分层

| 层级 | 可见范围 | 示例 |
|------|----------|------|
| **个人草稿** | 仅创建者 | 萃取中、未提交 |
| **部门库** | 本部门 | 「财务-报销审批」「客服-退款处理」 |
| **公司公共库** | 全员只读 | 「信息安全-数据分级」「通用-会议纪要」 |
| **归档库** | 管理员 | 过期、被 superseded 的技能 |

### Phase 0：试点验证（第 1–2 月）

**目标**：2 个部门、30 人、≥ 20 个经确认的技能，验证业务价值。

#### 0.1 选型

| 候选部门 | 理由 | 推荐 |
|----------|------|:----:|
| 客服/运营 | 流程标准化强、ROI 直观 | ★ |
| 财务 | 审批规则清晰 | ★ |
| 研发 | 工具链成熟（Cursor） | 辅助 |

#### 0.2 试点配置

- 部署：单实例 Docker（内网）+ 公司 DeepSeek/私有 LLM 网关
- 通道：飞书群 2 个 + Cursor 10 人
- 制度：Champion 每周审核待确认声明；重复技能合并

#### 0.3 交付物

- [ ] 《试点运行手册》（话术见 `docs/USER_GUIDE.md`）
- [ ] 试点周报模板（创建数、验证率、问题清单）

| 用户 NPS（试点组） | ≥ 7/10 |
| 信安评审 | 有条件通过或已通过 |

### Phase 1：企业平台化（第 3–4 月）

**目标**：补 P0 缺口，支撑 5 个部门同时使用。

#### 1.1 产品/engineering  backlog

| # | 工作项 | 说明 | 估时 |
|---|--------|------|------|
| P1-1 | **部门租户隔离** | `SKILLOS_SKILLS_DIR/{dept_id}/` + API 按部门过滤 | 2 周 |
| P1-2 | **飞书 SSO** | 飞书 OAuth → user_id/dept_id 注入 session | 2 周 |
| P1-3 | **飞书企业机器人** | 官方 webhook，免 Hermes 手工配置；消息 → dispatch | 3 周 |
| P1-4 | **技能审批流** | 草稿 → 待审 → 已发布 → 归档；对接 SkillHub RBAC | 2 周 |
| P1-5 | **技能门户 Web** | 列表/搜索/详情/待确认声明一键确认 | 3 周 |
| P1-6 | **去重与命名** | 创建时相似技能提示；强制 `{dept}-{name}` | 1 周 |
| P1-7 | **审计日志** | 谁创建/修改/发布/调用；导出 CSV | 1 周 |
| P1-8 | **快速模式** | 一次长描述 → 直接 CONFIRMING（少轮对话） | 1 周 |

#### 1.2 基础设施

| 项 | 规格 |
|----|------|
| 部署 | K8s 或 2 节点 HA（API + MCP 分离） |
| 数据库 | PostgreSQL（用户/审计/审批）；技能文件仍用 Git 或对象存储 |
| LLM | 公司统一 API 网关；按部门配额 |
| 备份 | 技能库每日备份；RPO 24h / RTO 4h |

#### 1.3 信息安全

- [ ] 数据分级矩阵（哪些字段禁止进 LLM）
- [ ] 对话脱敏（手机号、身份证、金额自动 mask）
- [ ] 私有化选项评估（本地 Ollama vs 专有云 DeepSeek）
- [ ] 渗透测试（API 鉴权、越权访问）

#### 1.4 验收

- [ ] 5 部门并行，无 cross-tenant 泄漏（渗透测试通过）
- [ ] 飞书机器人在 3 分钟内响应 95% 消息
- [ ] 审批流端到端跑通

### Phase 2：全面推广（第 5–8 月）

**目标**：全部业务部门接入，≥ 200 个已发布技能。

#### 2.1 推广节奏

| 批次 | 部门 | 时间 |
|------|------|------|
| Batch 1 | 试点 2 部门扩编 | M5 |
| Batch 2 | HR、行政、采购 | M6 |
| Batch 3 | 销售、市场、产品 | M7 |
| Batch 4 | 剩余部门 | M8 |

#### 2.2 培训体系

| 对象 | 内容 | 形式 |
|------|------|------|
| 全员 | 15 分钟「什么是 Skill、如何一句话沉淀」 | 飞书微课 |
| Champion | 2 小时工作坊：审核、去重、认识论确认 | 线下/直播 |
| IT | 运维、监控、故障处理 | 文档 + 演练 |

#### 2.3 运营机制

- **月度技能榜**：各部门创建数、verified 率、调用量
- **季度技能评审会**：晋升优质技能到公司公共库
- **「技能悬赏」**：业务痛点发榜，Champion 带队沉淀

#### 2.4 集成（可选，按优先级）

| 集成 | 价值 |
|------|------|
| 飞书文档 / 语雀 | 从现有 SOP 一键 import |
| Jira / 工单 | 技能与流程工单关联 |
| 新人 onboarding | 部门技能包一键订阅 |

#### 2.5 验收

| 指标 | 目标 |
|------|------|
| 部门覆盖率 | 100% 有 Champion |
| 已发布技能 | ≥ 200 |
| 月活创建用户 | ≥ 15% 员工 |
| 平均萃取耗时 | ≤ 8 轮对话或快速模式 1 次 |

### Phase 3：治理与合规深化（第 9–12 月）

**目标**：可持续运营，满足审计要求。

#### 3.1 治理政策（需 HR/法务会签）

1. **技能 ownership**：每个已发布技能必须有在职 owner，离职需交接  
2. **review 周期**：部门库技能每 6 个月复审；规则变更需新版本  
3. **禁止事项**：未脱敏客户数据、未授权商业机密写入技能  
4. **争议处理**：业务规则冲突由部门负责人 + Reviewer 裁定  

#### 3.2 技术增强

| 工作项 | 说明 |
|--------|------|
| 技能版本 diff | 对比 v1/v2，高亮规则变更 |
| 失效与 superseded | 旧规则标记 invalid_at，Agent 不引用 |
| 成本中心分摊 | 按部门 LLM token 用量报表 |
| SLA 监控 | Prometheus + Grafana；告警到 IT 飞书群 |
| 灾备演练 | 每季度一次恢复演练 |

#### 3.3 验收

- [ ] 内外部审计抽样：100% 技能可追溯 creator + 审批链
- [ ] verified 率 ≥ 70%
- [ ] 零 P0 安全事故

---

### Phase 4：智能化与持续优化（第 13–18 月）

**目标**：越用越好，形成公司知识飞轮。

| 方向 | 内容 |
|------|------|
| **推荐** | 根据岗位/部门推荐应订阅的技能 |
| **质量评分** | 自动检测过时规则（与 Playbook 冲突） |
| **MetaSkill** | 多技能编排（如「退款→CRM→旺旺」流水线） |
| **SkillOpt 对接** | `export_for_skillopt` 闭环优化 |
| **多语言** | 涉外部门英文 Skill 模板 |

## 六、技能生命周期（标准流程）

```
员工对话/粘贴 SOP
       │
       ▼
  ┌─────────┐
  │ 草稿    │ ← 个人可见，可继续编辑
  └────┬────┘
       │ 提交
       ▼
  ┌─────────┐
  │ 待审核  │ ← Champion 检查事实 + 认识论 pending
  └────┬────┘
       │ 批准
       ▼
  ┌─────────┐
  │ 部门库  │ ← 本部门 Agent 可调用
  └────┬────┘
       │ 季度评审 / 悬赏晋升
       ▼
  ┌─────────┐
  │ 公司库  │ ← 全员只读
  └────┬────┘
       │ 过时 / 被替代
       ▼
  ┌─────────┐
  │ 归档    │
  └─────────┘
```

**认识论确认嵌入点**：待审核阶段，Champion 必须处理全部「硬规则」pending 项方可发布。
