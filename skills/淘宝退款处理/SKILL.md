# 技能名称：淘宝退款处理
## 核心问题
规范处理淘宝小店每日约200单的仅退款/退货退款申请，确保按政策规则执行并记录完整。

## S_body
1. 核对订单信息：在旺旺或订单页获取订单号、实付金额、付款渠道（支付宝/微信），在ERP中查询发货状态和物流签收时间。
2. 判断退款类型与处理：
   - if 未发货 → 执行秒退（无需额外审批）。
   - if 已发货且未签收 → 联系快递拦截，等待拦截结果后处理退款。
   - if 已签收 → 走七天无理由流程，检查商品完好性；若签收超过七天，转主管审批。
3. 金额与权限检查：
   - if 实付金额超过500元 → 转主管审批，并在飞书群留截图。
   - if 金额≤500元且签收未超七天 → 按正常流程处理。
4. 边界情况处理：
   - if 买家威胁差评 → 升级至主管，但不默认全额退款。
   - if 虚拟商品（下载链接类）且已发货 → 不支持退款。
   - if 换货请求 → 不属于退款，走另一个工单类型。
5. 输出物记录：
   - 在ERP中标记退款原因分类（质量问题/不喜欢/发错货）。
   - 退款完成后24小时内，通过旺旺发送一条模板消息确认。
   - 同步更新CRM备注，记录退款详情。

## S_params
- 订单号: string, 无默认值, 必填，退款申请对应的订单编号
- 实付金额: float, 无默认值, 必填，订单实际支付金额（元）
- 付款渠道: string, 默认值 "支付宝", 可选值 "支付宝" 或 "微信"
- 发货状态: string, 默认值 "未发货", 可选值 "未发货" / "已发货" / "已签收"
- 物流签收时间: datetime, 无默认值, 可选，仅当发货状态为"已签收"时必填
- 退款原因分类: string, 默认值 "不喜欢", 可选值 "质量问题" / "不喜欢" / "发错货"
- 是否威胁差评: boolean, 默认值 false, 标记买家是否以差评威胁
- 是否虚拟商品: boolean, 默认值 false, 标记是否为下载链接类虚拟商品
