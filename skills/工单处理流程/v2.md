# 工单处理流程

## 核心问题
这个流程解决如何从接收客户工单到最终关单的全过程，包括需求转交、SLA 响应、信息补全、升级二线、关单确认和严重故障通知。

## When to use
- keywords: 工单, 故障, 支持, 问题, 报修, 投诉, 需求, 升级, 关单, SLA, P1, P2, P3, INC-
- context: 当用户需要处理客户支持工单、故障报修、功能需求或需要了解工单处理流程时。
- excludes: 当用户询问产品使用方法、一般性咨询、或与工单处理流程无关的对话时，不要使用此技能。

## Instructions
Follow these steps in order. Ask the user if anything is marked [待确认].

1.  接收客户工单或请求。
2.  判断请求类型：
    -   如果是**功能请求**（Feature Request），记录需求详情并转交产品团队处理。**不计入故障 SLA**。
    -   如果是**故障/问题报告**，进入下一步。
3.  判断故障严重等级（P1/P2/P3）：
    -   **P1（严重故障）**：4 小时内首次响应。**必须同步通知客户经理和客户成功团队**。
    -   **P2（一般故障）**：当天首次响应。
    -   **P3（轻微问题）**：三个工作日内首次响应。
4.  [新增] **判断是否为重复工单**：
    -   如果客户明确表示“重复工单”或与已有工单主题、客户、产品版本完全一致，则**合并到母单**。
    -   合并时，**保留所有时间线**（包括子单的创建、响应、升级记录），母单状态不变。
    -   如果无法确认，则视为新工单继续处理。
5.  检查客户提供的信息是否完整（如日志、截图、产品版本号）：
    -   如果信息不全，使用模板邮件向客户索取缺失信息（日志、截图、产品版本号）。
    -   如果信息完整，进入下一步。
6.  尝试根据现有信息进行初步排查和解决。
7.  如果问题无法解决，需要升级到二线支持：
    -   升级前必须确保附带**完整对话记录、已尝试的解决步骤、环境信息**。
8.  在工单处理过程中，可以创建 Jira 缺陷单进行双向关联。工单号格式为 `INC-` 开头。
9.  在准备关单前，**必须获得客户确认问题已解决**。
    -   如果客户确认已解决，关闭工单。
    -   如果客户未确认或问题未解决，保持工单状态为 `open`，并继续处理。
