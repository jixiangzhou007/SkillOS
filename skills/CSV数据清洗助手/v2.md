# 我要一个 CSV

## 核心问题
解决运营同学从销售系统导出的CSV文件中常见的重复行、空邮箱、金额格式错误、日期格式不统一等问题，输出干净可用的数据表和异常记录表。

## When to use
- keywords: CSV清洗, 数据清洗, 去重, 重复行, 空邮箱, 金额格式, 日期格式, 清洗报告, 销售导出, 运营数据, 异常表导出, 整理数据
- context: 用户上传或提及需要处理CSV文件，且目标是“清洗”、“去重”、“整理”或“格式化”数据。
- excludes: 用户仅要求“查看”、“分析”、“统计”或“可视化”数据，未提及清洗需求时，不触发此技能。

## Instructions
Follow these steps in order. Ask the user if anything is marked [待确认].

1.  **读取与探测**：读取用户上传的CSV文件，自动探测文件编码（优先UTF-8，失败则尝试GBK）和分隔符（逗号、制表符等）。加载前5行作为样本，识别列名和数据类型（文本、数字、日期、邮箱等）。如果文件编码无法识别或出现乱码，提示用户重新导出为UTF-8格式。
2.  **按主键去重并处理邮箱异常**：
    -   以 `id` 列作为主键进行去重。如果存在多行具有相同 `id`，则保留创建时间（或行号）最早的一行，删除其余重复行，并记录被删除的重复行数量。
    -   **if** 文件中不存在 `id` 列，则 **then** 中止流程，并提示用户“未找到主键列 `id`，请确认文件包含该列”。
    -   扫描 `email` 列，识别以下情况为“异常行”：值为空（NULL或空字符串）、格式不合法（如不含 `@` 符号、包含空格、域名部分不标准等）。将所有异常行（包含原始所有列）提取出来，并新增一列 `error_reason`，注明具体错误原因（如“空值”、“缺少@”）。将这些异常行导出为 `error_<原文件名>.csv`。**不静默删除**：在清洗后的主表中，将异常行的 `email` 列内容替换为“详见异常表”，保留该行的其他数据。
3.  **金额格式规范化与日期格式统一**：
    -   扫描所有金额列（通过列名如“金额”、“price”、“amount”或数据类型识别）。去除列中的货币符号（如 `￥`、`$`、`€`）和千位分隔符（`,`），将清理后的值转换为 `decimal` 类型。**负数标红**：如果转换后的值为负数，在清洗报告中标注该值，并提示用户。**if** 用户确认需要输出带样式的CSV，则 **then** 在生成的CSV中将负数单元格背景标红；**else** 仅在报告中列出负数所在行号和值。**格式异常处理**：如果某单元格包含无法解析为数字的文本（如“约100元”），则将该行标记为“格式异常”，并移入异常表（`error_<原文件名>.csv`），在清洗后的主表中该单元格内容替换为“格式异常”。
    -   扫描所有日期列（通过列名如“日期”、“date”、“time”或数据类型识别）。尝试将各种常见日期格式（如 `2023/01/01`、`01-01-2023`、`2023年1月1日`）统一转换为 `YYYY-MM-DD` 格式。在清洗后的文件中，为每个被转换的日期列旁边新增一列 `原值_<列名>`，记录转换前的原始值，供用户核对。**无法解析的日期**：如果某单元格无法被任何已知格式解析，则将该行标记为“日期格式异常”，移入异常表，在清洗后的主表中该单元格内容替换为“日期异常”。
4.  **生成清洗报告与输出文件**：
    -   生成一份清洗报告，包含以下指标：清洗前总行数、清洗后总行数、去重数量、异常行数（邮箱+金额+日期异常的总和）、各关键字段（如 `email`、`金额`、`日期`）的填充率（非空值行数 / 总行数 * 100%）、负数金额列表（如有）。
    -   生成清洗后的主数据文件，命名为 `cleaned_<原文件名>.csv`。
    -   将清洗报告以文本形式展示给用户。

## Decision routes
| 用户意图/条件 | 执行动作 | 备注 |
|------------|---------|------|
| 用户上传CSV并说“清洗数据” | 执行完整清洗流程（步骤1-4） | 默认流程 |
| 文件中缺少 `id` 列 | 中止流程，提示用户“未找到主键列 `id`” | 无法自动处理 |
| 文件编码无法识别 | 提示用户“编码错误，请重新导出为UTF-8格式” | 需要用户介入 |
| 用户要求“只看重复行” | 仅执行步骤2中的去重部分，并输出重复行列表 | 非完整清洗 |
| 用户要求“只检查邮箱” | 仅执行步骤2中的邮箱检测部分，并输出异常表 | 非完整清洗 |

## Inputs
- file_path: string, 无默认值, 用户上传的CSV文件路径。
- dedupe_key: string, 默认值 "id", 用于去重的主键列名。
- output_dir: string, 默认值 "当前目录", 输出文件的保存目录。

## Outputs
- status: enum — 执行结果（success / pending / rejected / escalated）
- message: string — 给用户或下游系统的摘要说明
- audit_notes: string — 关键操作记录（可选）
