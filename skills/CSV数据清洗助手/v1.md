# CSV数据清洗助手

## 核心问题
运营同学拿到销售导出的CSV后，手动清洗重复行、空邮箱、金额格式和日期格式太耗时，需要一键自动化处理。

## When to use
- keywords: csv, 清洗, 数据清洗, 去重, 重复行, 空邮箱, 金额格式, 日期格式, 销售导出, 运营数据
- context: 运营同学处理销售导出的CSV文件时，需要自动清理脏数据
- excludes: 用户要求处理Excel文件（.xlsx/.xls）、用户要求做数据可视化或图表、用户要求做数据统计分析（如回归、预测）

## Instructions
Follow these steps in order. Ask the user if anything is marked [待确认].

1. **读取CSV文件**
   - 接收用户提供的 `csv_path` 参数
   - 自动检测文件编码（UTF-8/GBK）和分隔符（逗号/制表符）
   - 若文件不存在或无法解析，返回错误提示：“无法读取文件，请检查路径和文件格式”
   - 记录总行数（不含表头）

2. **去重处理**
   - 根据 `dedup_columns` 参数指定的列（默认全列）识别重复行
   - 保留最早一条（按行号顺序），删除后续重复行
   - 记录被删除的行号及内容到日志
   - 累加去重计数

3. **邮箱字段检查**
   - 根据 `email_column` 参数（默认"email"）定位邮箱列
   - 对每一行检查：是否为空，或是否不含“@”和“.”
   - 若异常：将该行复制到异常表（`error_output_path`），在主表中将该字段标记为“待补充”
   - 累加邮箱异常计数

4. **金额格式标准化**
   - 根据 `amount_column` 参数（默认"amount"）定位金额列
   - 对每一行执行以下清洗：
     - 去除货币符号（¥、$、€）
     - 去除千位分隔逗号
     - 若 `amount_unit_conversion` 为true，自动转换汉字单位（如“万元” → 乘以10000，“千元” → 乘以1000）[待确认：单位映射表]
     - 若 `amount_negative_highlight` 为true，对负数金额在输出CSV中通过字体颜色标记为红色
   - 若无法解析为数字，将该行复制到异常表，在主表中标记为“金额格式错误”

5. **日期统一格式化**
   - 根据 `date_column` 参数（默认"date"）定位日期列
   - 自动识别常见日期格式（如 `2024/3/15`、`2024年3月15日`、`03-15-2024` 等）
   - 统一转换为 `date_output_format` 指定的格式（默认"YYYY-MM-DD"）
   - 若无法解析，将该行复制到异常表，在主表中标记为“日期格式错误”

6. **生成清洗报告**
   - 输出以下指标：
     - 总行数（原始）
     - 去重数（删除的重复行数）
     - 异常数（邮箱异常 + 金额异常 + 日期异常的总行数，去重）
     - 各字段填充率（email、amount、date列的非空占比）

7. **保存输出文件**
   - 主表保存到 `output_path`（若不指定，在原文件同目录下生成 `原文件名_cleaned.csv`）
   - 异常表保存到 `error_output_path`（若不指定，在原文件同目录下生成 `原文件名_errors.csv`）

## Decision routes
| 用户意图/条件 | 执行动作 | 备注 |
|------------|---------|------|
| 用户说“清洗这个CSV”或上传CSV文件 | 执行完整清洗流程（步骤1-7） | 使用默认参数，若用户未指定列名则用默认值 |
| 用户指定了去重列（如“按id和name去重”） | 在步骤2中使用 `dedup_columns=["id","name"]` | 逗号分隔的列名需与CSV表头完全一致 |
| 用户说“不要标红负数” | 在步骤4中设置 `amount_negative_highlight=false` | 负数金额正常显示，不标红 |
| 用户说“日期格式要带时间” | 在步骤5中设置 `date_output_format="YYYY-MM-DD HH:MM:SS"` | 仅当原始数据包含时间信息时有效 |
| 用户说“只看报告，不保存文件” | 执行步骤1-6，跳过步骤7 | 在对话中直接展示报告内容 |
| 用户说“帮我看看这个文件有什么问题” | 执行步骤1-5，输出异常行摘要和填充率 | 不生成清洗后的文件，仅做诊断 |

## Inputs
- `csv_path`: string，必需，CSV文件路径，支持绝对路径或相对路径
- `dedup_columns`: string，可选，默认""（全列去重），去重依据的列名列表，多个列名用逗号分隔（如 "id,name"）
- `email_column`: string，可选，默认"email"，邮箱字段的列名，必须与CSV表头完全一致
- `amount_column`: string，可选，默认"amount"，金额字段的列名，必须与CSV表头完全一致
- `date_column`: string，可选，默认"date"，日期字段的列名，必须与CSV表头完全一致 [待确认：用户确认列名]
- `output_path`: string，可选，默认自动生成，清洗后文件保存路径，若不指定则生成在原文件同目录下，文件名加 "_cleaned" 后缀
- `error_output_path`: string，可选，默认自动生成，异常数据输出路径，若不指定则生成在原文件同目录下，文件名加 "_errors" 后缀
- `amount_negative_highlight`: boolean，可选，默认true，是否对负数金额标红（在输出的CSV中通过字体颜色标记）
- `amount_unit_conversion`: boolean，可选，默认true，是否自动转换汉字单位（如“万元” → 乘以10000，“千元” → 乘以1000）[待确认：单位映射表]
- `date_output_format`: string，可选，默认"YYYY-MM-DD"，日期输出的目标格式，支持 "YYYY-MM-DD" 或 "YYYY-MM-DD HH:MM:SS"

## Outputs
- status: enum — 执行结果（success / pending / rejected / escalated）
- message: string — 给用户或下游系统的摘要说明
- audit_notes: string — 关键操作记录（可选）
