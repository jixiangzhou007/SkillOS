# 工单分诊与SLA管理

## 核心问题
这个流程解决如何根据客户提交的工单（邮件或Zendesk）自动识别类型、执行对应处理路径、遵守SLA响应时间，并在关单前获得客户确认。

## When to use
- keywords: 工单, 故障,

## Instructions
Follow these steps in order. Ask the user if anything is marked [待确认].

1. 识别工单类型：读取工单内容，判断属于以下三类之一：
   - 故障（系统不可用、报错、功能异常）
   - 咨询（如何使用、配置、最佳实践）
   - 功能请求（希望新增或修改功能）
2. 如果工单类型为**故障**：
   - 2a. 在 Confluence 知识库中搜索已知解决方案或类似故障记录。
   - 2b. 如果在 30 分钟内找到匹配方案，直接回复客户。
   - 2c. 如果 30 分钟内未找到方案，将工单升级至二线工程师团队，并在工单中注明“升级二线”。
3. 如果工单类型为**咨询**：
   - 3a. 在 FAQ 库中匹配最接近的问题与答案。
   - 3b. 计算匹配度。如果匹配度 >= 80%，直接回复 FAQ 内容。
   - 3c. 如果匹配度 < 80%，将工单转接至人工客服处理。
4. 如果工单类型为**功能请求**：
   - 4a. 记录客户需求详情（功能描述、期望用途、优先级）。
   - 4b. 将需求转交至产品团队。
   - 4c. 注意：功能请求不计入故障 SLA，无需响应时间承诺。
5. 信息补全：如果工单信息不完整（缺少日志、截图、产品版本号等），使用标准模板邮件向客户索取缺失信息。模板内容：[待确认具体模板文本]。
6. SLA 响应时间管理：
   - P1（严重故障）：首次响应必须在 4 小时内。
   - P2（一般故障）：首次响应必须在当天内。
   - P3（低优先级）：首次响应必须在 3 个工作日内。
   - 根据工单优先级设置对应的响应计时器或提醒。
7. 关单确认：在准备关闭工单前，必须向客户发送确认请求，询问问题是否已解决。
   - 如果客户回复“已解决”或确认，则关闭工单。
   - 如果客户未确认或回复“未解决”，保持工单状态为 Open，并继续处理。
