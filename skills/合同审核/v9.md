# 合同审核

## 核心问题
快速识别销售合同中的关键风险条款，给出修改建议和谈判要点。

## When to use
- keywords: contract, 合同, 审核, 条款, 风险, 销售合同
- context: 用户上传合同文件或提出合同审核需求时
- excludes: 非合同类文件审核、法律诉讼咨询、合同起草（非审核）

## Instructions
Follow these steps in order. Ask the user if anything is marked [待确认].

1. 检查文件名是否包含 "contract" 关键词
   - 如果文件名包含 "contract"，则自动触发合同审核流程
   - 如果文件名不包含 "contract"，询问用户是否仍要执行合同审核

2. 识别合同类型和基本结构
   - 确认是否为销售合同（非采购、租赁、保密等其他类型）
   - 如果非销售合同，提示用户本技能专为销售合同设计，询问是否继续

3. 逐项审核以下核心条款：
   a. 价格条款：检查定价方式、付款条件、价格调整机制、税费承担
   b. 质保条款：检查质保期限、质保范围、质保责任限制
   c. 责任条款：检查赔偿限额、免责条款、违约金标准、责任上限
   d. 知识产权条款：检查知识产权归属、使用许可、侵权赔偿
   e. 违约条款：检查违约情形、违约责任、解除条件

4. 对每个条款输出审核结果：
   - 风险等级（高/中/低）
   - 风险描述
   - 修改建议
   - 谈判底线 [待确认：用户是否有预设的谈判底线标准]

5. 生成合同审核报告
   - 汇总所有风险条款
   - 按风险等级排序
   - 给出整体评估结论
   - 建议下一步行动

## Decision routes
| 用户意图/条件 | 执行动作 | 备注 |

| 用户上传文件名含 "contract" 的文件 | 自动触发完整合同审核流程 | 核心触发场景 |
| 用户上传文件名不含 "contract" 但要求审核合同 | 确认后执行完整审核流程 | 手动触发 |
| 用户上传非销售合同文件 | 提示本技能专为销售合同设计，询问是否继续 | 边界情况处理 |

## Inputs
- file_path: string, 无默认值，待审核合同文件的路径
- contract_type: string, 默认值 "sales"，合同类型（sales/purchase/other）
- risk_threshold: string, 默认值 "medium"，风险关注阈值（high/medium/low）
- output_format: string, 默认值 "report"，输出格式（report/summary/checklist）

## Outputs
- status: enum — 执行结果（success / pending / rejected / escalated）
- message: string — 给用户或下游系统的摘要说明
- audit_notes: string — 关键操作记录（可选）
