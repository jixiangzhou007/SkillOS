# 合同审核

## 核心问题
快速识别销售合同中的关键风险条款，输出审核意见，降低签约风险。

## S_trigger
- keywords: contract, 合同, 审核, 审查, 条款, 风险, 销售合同
- context: 用户上传文件或输入文本，且文件名包含 "contract" 或用户明确要求审核合同
- excludes: 非销售合同（如采购合同、劳动合同、租赁合同）；用户仅要求翻译合同而非审核

## S_body
Follow these steps in order. Ask the user if anything is marked [待确认].

1. 接收合同审核需求：
   - 触发条件：用户上传文件或输入合同文本，且文件名包含 "contract" 关键词
   - 合同来源：销售合同
   - 若文件名不包含 "contract" 但用户明确要求审核合同，则询问用户是否确认该文件为销售合同
2. 合同初步审查：
   - 提取合同中的以下关键信息：[待确认]（如合同双方、签约日期、合同金额、产品/服务描述）
   - 检查合同是否包含以下必要条款：[待确认]（如价格条款、交付条款、付款条款、违约责任）
   - 若缺少必要条款，记录缺失项并继续审查
3. 风险条款识别：
   - 逐条审查以下高风险领域：
     - 价格条款：是否存在价格不明确、折扣条件模糊、价格调整机制缺失等问题
     - 保修条款：保修期限、范围、免责条件是否合理
     - 责任条款：赔偿上限、责任免除、间接损失排除是否对己方不利
     - 知识产权条款：知识产权归属、使用许可范围、侵权赔偿是否清晰
     - 违约条款：违约金比例、违约情形定义、解除合同条件是否公平
   - 若发现风险条款，标记为「高风险」「中风险」或「低风险」，并引用具体条款编号
4. 审核意见输出：
   - 输出格式：结构化报告，包含以下部分：
     - 合同基本信息摘要
     - 风险条款清单（按风险等级排序）
     - 修改建议（针对每个风险条款）
     - 总体审核结论（通过/有条件通过/不通过）
   - 若总体审核结论为「不通过」，需说明核心障碍和谈判底线

## S_route
| 用户意图/条件 | 执行动作 | 备注 |
|------------|---------|------|
| 用户上传文件且文件名含 "contract" | 自动触发合同审核流程 | 默认触发方式 |
| 用户上传文件但文件名不含 "contract"，用户明确要求审核合同 | 询问确认是否为销售合同，确认后触发审核 | 避免误触发 |
| 用户直接输入合同文本并要求审核 | 直接进入步骤2合同初步审查 | 跳过文件解析步骤 |
| 用户要求仅审查特定条款（如只看违约责任） | 跳过步骤2，直接进入步骤3中对应条款的审查 | 按需聚焦 |
| 审核过程中发现合同为框架协议而非具体销售合同 | 提示用户该技能专为销售合同设计，询问是否继续 | 边界情况处理 |

## S_params
- contract_source: string, 默认值 "sales", 可选值 ["sales", "other"]，合同来源类型
- review_depth: string, 默认值 "full", 可选值 ["full", "quick", "clause-specific"]，审核深度
- risk_threshold: string, 默认值 "medium", 可选值 ["low", "medium", "high"]，风险报告的最低风险等级
- output_format: string, 默认值 "structured", 可选值 ["structured", "summary", "full"]，输出格式

## Outputs
- status: enum — 执行结果（success / pending / rejected / escalated）
- message: string — 给用户或下游系统的摘要说明
- audit_notes: string — 关键操作记录（可选）
