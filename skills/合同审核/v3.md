# 技能名称：合同审核
## 核心问题
对合同文本进行系统性审查，识别并评估条款风险，输出修改建议，确保合同权益与合规性。

## 前置条件
- 用户需提供待审核的合同文件（PDF/Word/文本）。
- 若用户有内部标准模板或历史版本，建议一并提供，以便进行条款比对。
- **新增**：若合同涉及价格、数量、交付等核心商业条款，建议用户同时提供报价单、订单或谈判纪要作为参考。

## 主要流程（S_route）
| 步骤 | 动作 | 说明 |
| :--- | :--- | :--- |
| 1. 条款比对 | 比对差异 | 将待审合同与用户提供的标准模板、历史版本或行业示范文本（如市场监管总局合同示范文本库）进行对比，高亮所有修改、新增或删除的条款。 |
| 2. 风险识别 | 分类与定级 | 对每处差异进行法律与商业风险评估，按以下等级分类：<br>- **高风险**：可能导致重大经济损失或法律败诉，必须修改；<br>- **中风险**：存在不利隐患或模糊地带，建议修改或澄清；<br>- **低风险**：仅为表述或格式差异，不影响实质权利义务，仅标注。 |
| 3. 修改建议 | 输出方案 | 针对每处风险点，提供具体的修改措辞、法律依据或谈判策略，并标注修改优先级。 |

## 边界情况处理
- **未提供模板**：若用户无参考模板，则基于合同类型（如采购、技术服务、租赁）调用行业通用风险清单进行审查，并在输出中注明“未进行模板比对”。
- **合同版本混乱**：若用户提供多个版本且未标注版本号，要求用户先明确基准版本，或自动按文件修改时间排序。
- **非标准语言**：若合同使用非中文或含大量专业术语，需先确认用户是否需要翻译或术语解释。

## 输出格式
输出应为结构化报告，包含：
1. 比对概览（差异总览表）
2. 风险清单（按高/中/低排序，含具体条款编号）
3. 修改建议（含推荐措辞与理由）
4. 争议解决提示（如管辖法院/仲裁机构、适用法律变更等）

## 新增：销售合同专项审查要点（增量）
### 第一步：内容与价格审查（用户指定流程起点）
- **动作**：重点审查合同正文及附件中的**价格条款、数量条款、计价方式、付款条件、价格调整机制**。
- **风险点示例**：
  - **隐蔽价格调整**：如“最低采购量未达标则单价上浮”、“原材料价格波动超过X%则重新定价”等，需评估是否与主价格条款矛盾或隐藏成本。
  - **数量与总价不一致**：检查附件清单数量、单价与合同正文总价是否匹配。
  - **付款条件模糊**：如“验收后付款”未明确验收标准与时限。
- **风险等级建议**：
  - **高风险**：价格调整机制与主价格条款直接冲突，或调整条件极易触发且幅度过大（如 > 20%）。
  - **中风险**：价格调整机制存在，但触发条件较严格或幅度较小，建议明确或争取删除。
  - **低风险**：仅为表述不清（如“约100元”），可标注澄清。
