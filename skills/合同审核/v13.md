# 合同审核

## 核心问题
确保销售合同中的关键商业和法律条款符合我方利益，识别并标记风险条款。

## S_trigger
- keywords: contract, 合同, 审核, 条款, 风险, 销售合同
- context: 用户上传或提及销售合同文件，要求进行条款审核或风险评估
- excludes: 非销售合同（如采购合同、租赁合同、劳动合同）；用户仅要求翻译合同而非审核；用户要求起草合同而非审核

## S_body
Follow these steps in order. Ask the user if anything is marked [待确认].

1.  **读取合同文件**：获取用户提供的合同文件内容。如果文件无法读取或为空，则输出错误信息："无法读取合同文件，请确认文件格式正确且非空"。
2.  **提取关键条款**：从合同文本中提取以下五个维度的条款内容：
    -   价格与支付条款（含金额、币种、支付节点、支付方式）
    -   质保条款（含质保期限、质保范围、质保责任上限）
    -   责任限制条款（含赔偿上限、免责情形、间接损失排除）
    -   知识产权条款（含知识产权归属、使用许可、侵权赔偿）
    -   违约条款（含违约金计算方式、违约情形、解除权）
    -   若某个维度在合同中未明确提及，则标记为"未约定"。
3.  **条款风险分析**：对每个提取的条款维度进行风险判断：
    -   **价格与支付**：若支付节点晚于交付后60天，或存在"背对背"支付条款，标记为"高风险"；若支付节点在30-60天内，标记为"中风险"；否则标记为"低风险"。
    -   **质保条款**：若质保期超过3年，或质保范围包含"所有故障"无例外，标记为"高风险"；若质保期在1-3年，标记为"中风险"；否则标记为"低风险"。
    -   **责任限制**：若赔偿上限低于合同总价的50%，或排除了"根本违约"责任，标记为"高风险"；若赔偿上限在50%-100%之间，标记为"中风险"；否则标记为"低风险"。
    -   **知识产权**：若约定开发成果的知识产权归客户所有，或未约定我方使用客户背景IP的许可，标记为"高风险"；若约定共同所有但未明确行使规则，标记为"中风险"；否则标记为"低风险"。
    -   **违约条款**：若违约金超过合同总价的30%，或仅约定了单方违约责任，标记为"高风险"；若违约金在10%-30%之间，标记为"中风险"；否则标记为"低风险"。
4.  **生成审核报告**：将上述分析结果整理为结构化报告，包含：
    -   合同基本信息（文件名、文件类型）
    -   各维度条款原文摘要
    -   各维度风险等级（高/中/低）及风险说明
    -   综合风险评级：若存在2个及以上"高风险"维度，综合评级为"高风险"；存在1个"高风险"或3个及以上"中风险"，综合评级为"中风险"；否则为"低风险"。
    -   修改建议：针对每个"高风险"和"中风险"条款，给出具体的修改建议文本。

## S_route
| 用户意图/条件 | 执行动作 | 备注 |
|------------|---------|------|
| 用户上传合同文件，文件名包含"contract" | 执行S_body全部步骤，输出审核报告 | 自动触发 |
| 用户上传合同文件，文件名不包含"contract" | 询问用户是否仍要执行合同审核 | 避免误触发 |
| 用户明确要求审核合同条款 | 执行S_body全部步骤，输出审核报告 | 手动触发 |
| 用户询问合同风险或条款含义 | 执行S_body步骤2-4，输出针对性的风险分析 | 聚焦用户关注点 |
| 合同文件无法读取或为空 | 输出错误提示，要求重新提供有效文件 | 异常处理 |

## S_params
- contract_file: file, 必填，待审核的合同文件（支持PDF、Word、纯文本格式）
- focus_dimensions: array, 默认["price","warranty","liability","ip","penalty"]，指定需要重点审核的条款维度
- risk_threshold: string, 默认"medium"，输出报告的最低风险等级（"low"输出所有等级，"medium"仅输出中高风险，"high"仅输出高风险）

## Outputs
- status: enum — 执行结果（success / pending / rejected / escalated）
- message: string — 给用户或下游系统的摘要说明
- audit_notes: string — 关键操作记录（可选）
