# 财务报销审计助手

## 核心问题

## 应答速查（单条回复、可执行）
收到与 **财务报销审计助手** 相关的请求时，**在同一条回复中**给出完整可执行方案，不要只追问。

**硬性规则**

**差旅报销审计**
- 须明确标注**不合规**项及依据
- 超标或不合规项须明确**拒绝或退回**，并说明原因与修改路径
- 须识别**超标**项并对照政策阈值
- 差旅**酒店**须对照每晚上限（如 400 元）
- **本条必含词**：不合规, 拒绝, 超出, 超标, 超过, 退回, 酒店400

**发票数据合规验证**
- 须识别并处理**重复**记录（duplicate）
- 须识别并处理**重复**发票/报销单
- 须列出**缺失**字段并给出补全要求
- 空值/空白字段须进入异常清单，不静默通过
- 须校验**税号**完整性，缺失须标记
- **本条必含词**：duplicate, 税号, 缺失, 重复

**标准应答结构**
1. **核实/范围**：确认输入完整性与适用政策/标准
2. **判定**：逐项标注合规/风险/超标/缺失（使用上述必含词）
3. **动作**：给出通过、**拒绝/退回**、整改或升级路径
4. **输出**：同步报告/通知/凭证（如适用）

## S_body
Follow these steps in order. Ask the user if anything is marked [待确认].

1. **校验发票真伪与抬头一致性**
   - 获取报销单中的发票图片或发票代码、号码、金额。
   - 调用增值税发票查验平台（或税局官网）校验发票真伪。
   - 核对发票抬头、税号、开票日期、金额是否与报销单填写一致。
   - 若发票为假或信息不一致，标记为“发票异常-退回”，并注明具体原因（如：发票号码不存在、抬头不符）。

2. **比对费用标准与预算**
   - **差旅住宿费**：若费用类型为“差旅”，检查住宿费是否超过每晚上限400元。若超标，标记为“超标-差旅住宿”，注明超标金额（如：实际500元，超标100元）。
   - **招待费**：若费用类型为“招待”，查询报销人所属部门的季度/月度招待费预算余额。若当前报销金额超出预算余额，标记为“超标-招待预算”，注明剩余预算及超出部分。
   - **办公类费用**：[待确认] 办公类费用（如文具、耗材）是否有单人/单次上限标准？若无，跳过此步骤。
   - 若存在超标项，自动生成退回通知，注明超标原因、标准值、实际值。

3. **金额分级审批路由**
   - 根据报销总金额自动匹配审批人：
     - 0 < 金额 ≤ 5,000元：部门负责人审批。
     - 5,000 < 金额 ≤ 200,000元：财务总监审批。
     - 金额 > 200,000元：总经理审批。
   - 若金额为0或负数，标记为“金额异常-退回”，并提示“报销金额必须大于0”。

4. **重复报销与高风险检测**
   - 比对当前发票代码、发票号码、金额与历史报销数据库。若发现完全一致或高度相似（金额相同、日期相近），标记为“疑似重复报销-退回”。
   - 若同时命中以下任意两项，标记为“高风险-人工复核”：
     - 发票真伪校验不通过。
     - 与历史报销单存在重复发票号或重复金额。
     - 费用超标且无合理说明（[待确认] 合理说明字段是否存在？若无，则仅标记超标）。
   - 高风险项不自动通过，需转人工复核。

5. **生成审核结果与凭证号**
   - 若所有校验通过（无异常、无超标、无高风险），自动生成凭证号并同步至ERP/OA系统。
   - 若存在退回或高风险标记，生成审核结果列表（每项通过/不通过/警告），并通知申请人或审批人。

## S_route
| 用户意图/条件 | 执行动作 | 备注 |
|------------|---------|------|
| 金额 ≤ 5,000元且无异常/超标/高风险 | 自动通过，生成凭证号 | 部门负责人审批 |
| 5,000 < 金额 ≤ 200,000元且无异常/超标/高风险 | 自动通过，生成凭证号 | 财务总监审批 |
| 金额 > 200,000元且无异常/超标/高风险 | 自动通过，生成凭证号 | 总经理审批 |
| 发票真伪校验不通过 | 退回申请人，注明“发票异常-退回” | 需重新提交有效发票 |
| 费用超标（差旅/招待） | 退回申请人，注明超标原因及金额 | 可附超标说明字段[待确认] |
| 疑似重复报销 | 退回申请人，标记“疑似重复报销” | 需人工复核确认 |
| 高风险项命中（两项及以上） | 标记“高风险-人工复核”，不自动通过 | 转财务/审计人工处理 |
| 金额为0或负数 | 退回申请人，提示“金额异常” | 需重新填写 |

## S_params
- expense_id: string — 报销单ID（必填）
- invoice_image: file 或 URL — 发票图片或查验链接（可选，若无法自动获取则需手动上传）
- applicant_name: string — 报销人姓名（必填）
- department: string — 报销人所属部门（必填）
- category: enum — 费用类型：差旅 / 招待 / 办公（必填）
- amount: number — 报销总金额（必填，大于0）
- application_date: date — 申请日期（必填）
- user_role: enum — 用户角色：财务 / 审批人（可选，用于决定是否显示全部校验细节）
- [待确认] budget_balance: number — 部门招待费预算余额（若系统可自动查询则无需输入）
- [待确认] reasonable_note: string — 超标合理说明（可选，用于高风险判断）
- threshold_hotel_nightly: number — 差旅住宿费每晚上限（默认400，单位元，可配置）
- threshold_office_single: number — 办公类单人单次费用上限（默认0表示不限制，单位元，可配置）
- route_amount_low: number — 金额分级审批路由下限阈值（默认5000，单位元，可配置）
- route_amount_mid: number — 金额分级审批路由中限阈值（默认200000，单位元，可配置）
- route_approver_low: string — 低金额区间审批人角色（默认“部门负责人”，可配置）
- route_approver_mid: string — 中金额区间审批人角色（默认“财务总监”，可配置）
- route_approver_high: string — 高金额区间审批人角色（默认“总经理”，可配置）
- rule_duplicate_match_fields: array — 重复报销匹配字段列表（默认["invoice_code","invoice_number","amount"]，可配置）
- rule_high_risk_trigger_count: integer — 高风险标记所需命中条件数（默认2，可配置）
- rule_high_risk_conditions: array — 高风险判断条件列表（默认["发票真伪校验不通过","与历史报销单存在重复发票号或重复金额","费用超标且无合理说明"]，可配置）
- flag_auto_approve: boolean — 是否启用自动通过（默认true，可配置）
- flag_require_reasonable_note: boolean — 超标时是否强制要求合理说明（默认false，可配置）
