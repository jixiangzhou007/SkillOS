# 技术面试

## 核心问题
标准化技术岗位招聘面试流程，确保候选人从简历筛选到终面谈薪各环节高效、公平、可追溯。

## When to use
- keywords: 面试, 招聘, 简历筛选, 笔试, 技术面, CTO面, HR面, 谈薪, 候选人, 岗位JD
- context: HR或技术面试官在招聘系统中发起技术岗位面试流程时使用
- excludes: 非技术岗位招聘、实习生招聘流程、内部转岗面试

## Instructions
Follow these steps in order. Ask the user if anything is marked [待确认].

1. **简历筛选**
   - 输入：候选人简历PDF、岗位JD
   - 检查学历背景（是否满足硬性要求如211/985）与项目经验匹配度
   - 若学历不满足但项目经验高度匹配（大厂核心项目、对口技术栈、5年以上经验），标记为“待定”，安排5-10分钟快速电话预沟通确认项目真实性、技术深度及离职动机；预沟通通过后进入笔试环节
   - 若学历不满足且项目经验匹配度一般或存在夸大嫌疑，直接淘汰
   - 若发现简历时间线冲突（如项目时间与在职时间重叠）：
     - 轻度重叠（1个月内）：标记待定，先进入笔试，笔试通过后由HR进行背景核实
     - 明显重叠（3个月以上）或涉及核心项目经历造假嫌疑：直接淘汰，记录原因供复盘
   - 若项目描述与岗位要求显著不符：[待确认处理策略]
   - 通过筛选的候选人进入下一步

2. **在线笔试**
   - 通过牛客网发送笔试链接，限时90分钟
   - 题目构成：算法2题 + 系统设计1题
   - 系统自动判分，不及格则自动发送拒信
   - 及格则生成一面邀请
   - 若笔试平台异常：[待确认处理策略，如重试、人工代码审查或跳过笔试直接一面并加强考察]

3. **技术一面（面试官：你）**
   - 重点考察算法与系统设计能力
   - 打分1-5分，低于3分直接淘汰并自动发送拒信
   - 评分≥3分则自动安排下一轮面试
   - 面试后24小时内提交面评到招聘系统

4. **技术二面（面试官：CTO）**
   - 考察技术视野与团队文化匹配度
   - 评分≥3分则自动安排下一轮面试，低于3分自动发送拒信
   - 面试后24小时内提交面评到招聘系统

5. **HR终面**
   - 沟通薪资、福利与入职细节
   - 若候选人当前有竞业协议，HR需提前与候选人确认协议范围是否覆盖本公司业务领域，并记录确认结果

6. **候选人跳过环节请求处理**
   - 若候选人要求跳过笔试直接面试：[待确认处理流程]

| 笔试提交完成 | 触发自动判分逻辑（牛客网系统），不及格自动发拒信，及格生成一面邀请 | 笔试限时90分钟，算法2题+系统设计1题 |
| 一面/二面评价提交 | 触发面试结果判定：评分≥3分自动安排下一轮面试，<3分自动发送拒信并归档 | 面试后24小时内提交面评 |
| 候选人学历不满足但项目经验高度匹配 | 标记为“待定”，安排5-10分钟快速电话预沟通，通过后进入笔试 | 大厂核心项目、对口技术栈、5年以上经验 |
| 候选人学历不满足且项目经验匹配度一般 | 直接淘汰 | 避免后续环节资源浪费 |
| 候选人简历时间线轻度重叠（1个月内） | 标记待定，先进入笔试，笔试通过后由HR背景核实 | 不直接淘汰 |
| 候选人简历时间线明显重叠（3个月以上）或核心项目造假嫌疑 | 直接淘汰，记录原因供复盘 | 进入淘汰流程 |
| 候选人当前有竞业协议 | HR在HR终面前提前确认协议范围是否覆盖本公司业务 | 记录确认结果 |

## Inputs
- candidate_resume: file, 无默认值, 候选人简历PDF文件
- job_requirement: file, 无默认值, 岗位JD文本或PDF文件
- candidate_covenant_status: boolean, 无默认值, 候选人当前竞业协议状态，由HR在简历筛选阶段确认
- interview_score: integer, 无默认值, 面试评分1-5分，用于判定是否进入下一轮
-笔试_platform: string, 默认值"牛客网", 在线笔试平台名称
-笔试_duration: integer, 默认值90, 笔试限时（分钟）
-笔试_algorithm_questions: integer, 默认值2, 算法题数量
-笔试_system_design_questions: integer, 默认值1, 系统设计题数量
