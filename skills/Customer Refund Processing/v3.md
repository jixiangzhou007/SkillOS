# Skill Name: Customer Refund Processing
## Core Problem
Process a customer refund by verifying the order, checking policy, calculating the amount, and issuing payment.
## S_route

| Initiate refund | Verify order ID and purchase date | Order management system |

| Initiate refund | Process refund through payment gateway | Payment gateway API |
| Initiate refund | Send confirmation email to customer | Email notification system |
| Initiate refund | **Handle special cases (shipped not signed, over 7 days, amount > 500)** | **Order management / supervisor approval** |
| Initiate refund | **Handle boundary cases (threat of bad review, virtual goods, exchange request)** | **Customer service escalation / policy database** |
## S_body
1. Verify order ID and purchase date
2. Check refund policy (30-day window)
3. Calculate refund amount (product + tax - shipping)
4. **Check order shipping status:**
   - **If order has been shipped but not yet signed: contact logistics to intercept the package before proceeding.**
   - **If order has been signed for more than 7 days: escalate to supervisor for approval.**
5. **Check refund amount:**
   - **If refund amount exceeds 500 yuan: escalate to supervisor for approval.**
6. **Handle boundary cases:**
   - **If customer threatens to leave a negative review: escalate to customer service team for special handling.**
   - **If product is virtual goods (e.g., digital codes, services): do not support refund; inform customer and close request.**
   - **If customer requests exchange instead of refund: redirect to exchange/return work order process.**
7. Process refund through payment gateway
8. Send confirmation email to customer
