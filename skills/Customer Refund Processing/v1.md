# Skill Name: Customer Refund Processing
## Core Problem
Process customer refunds by verifying order eligibility, checking policy compliance, initiating payment reversal, and notifying the customer.
## S_route
| Intent | Action | Resource |
| Initiate Refund | Verify order in system | Order Management System |
| Check Policy | Validate return window, condition, fees | Refund Policy Database |
| Process Refund | Initiate refund in payment system | Stripe / Shopify Payments |
| Send Email | Send confirmation to customer | Email System |
## S_body
1. Verify the order in the system using the order ID, confirm customer name, items, purchase date, and amount paid, and ensure the order status is eligible (shipped/delivered, not already refunded).
2. Check the return policy: standard items 30 days, electronics 15 days; ensure item condition meets policy (unused, original packaging for electronics); check if restocking fee applies (15% for opened electronics, none for unopened); for digital goods, only refundable within 48 hours of purchase.
3. Process the refund in the payment system (Stripe / Shopify Payments) by selecting the original payment method, entering the refund amount (full or partial based on policy), adding an internal note with reason code (e.g., CUSTOMER_REQUEST, DEFECTIVE, WRONG_ITEM), and submitting to capture the refund transaction ID.
4. Send a confirmation email to the customer with refund amount, transaction/reference ID, and estimated time to appear on statement (3-5 business days for credit cards); BCC the support team for records; if partial refund, include explanation of restocking fee or deduction.
## S_trigger
- keywords: refund, return, customer refund, process refund, refund processing, refund order, refund policy, refund email
## S_params
- order_id: The unique identifier for the customer order to be refunded
- refund_amount: The amount to be refunded (full or partial based on policy)
- reason_code: The internal reason for refund (e.g., CUSTOMER_REQUEST, DEFECTIVE, WRONG_ITEM)
- refund_transaction_id: The unique transaction ID captured after processing the refund
