# 技能名称：技术支持工单处理

## S_body
1. 工单接收与分类
   - 信息核对：确认用户身份（免费版/企业版）、工单来源（邮件/在线/电话）、问题类型（配置/功能/故障）。
   - 优先级判定：
     - [待补充] 高优先级触发条件（如：影响核心业务、全站宕机、企业版客户）。
     - [待补充] 中优先级触发条件（如：功能异常但可绕过、配置报错）。
     - [待补充] 低优先级触发条件（如：咨询、非紧急优化建议）。
   - **边界情况处理**：
     - 若用户身份信息缺失（如匿名工单），则标记为“待验证”，不进入后续流程。
     - 若工单内容为空或为乱码，则自动回复要求补充问题描述。
     - **若用户身份明确（如企业版客户）但问题描述极简，缺乏关键信息（如无日志、无报错时间），则暂不提升优先级，先触发“信息收集流程”，再根据回复内容重新判定优先级。**

2. 问题诊断与排查
   - [待补充] 建立常见问题知识库（如：401报错 → 检查密钥权限/过期/IP白名单）。
   - [待补充] 引导用户提供关键信息（如：错误堆栈、请求时间戳、API调用示例）。
   - **边界情况处理**：
     - 若用户提供的日志/截图不完整，则发送标准化模板要求补充指定字段。
     - 若问题为已知Bug且已发布修复版本，则直接告知用户升级版本，无需深度排查。

3. 解决方案提供与执行
   - [待补充] 根据诊断结果，提供分步操作指南（可附带截图/视频链接）。
   - [待补充] 若需修改后台配置，需经用户确认后，由授权人员执行。
   - **边界情况处理**：
     - 若解决方案涉及敏感操作（如重置密钥），需二次验证用户身份（如：发送验证码到注册邮箱）。
     - 若用户拒绝推荐方案，则记录原因并升级至高级工程师。

4. 结果确认与关闭
   - [待补充] 主动询问用户问题是否解决，并请求评分（CSAT）。
   - [待补充] 若48小时内无用户反馈，则自动发送关闭提醒，3天后自动关闭工单。
   - **边界情况处理**：
     - 若用户反馈问题未解决，则自动将工单状态重置为“诊断中”，并通知原处理人。
     - 若用户要求关闭但问题未解决，需在工单备注中明确记录“用户主动要求关闭，问题未解决”。
