# 技能名称：技术支持工单处理

## 流程
1. **问题接收与分类**
   - 触发条件：客户通过邮件或 Zendesk 工单系统提交问题
   - 动作：识别问题类型（故障/咨询/功能请求）
   - 动作：根据影响范围和紧急程度，设定优先级（高/中/低）
   - **规则**：工单号格式为 INC- 开头，与 Jira 缺陷单可双向关联
   - **规则**：当工单标题与描述内容不一致（如标题为故障、描述含咨询），需按“标题优先”原则初步归类，并在工单内标记“存在混合需求”。
   - **规则**：对于信息不全的工单（如缺少日志、截图、产品版本号），自动发送模板邮件索取，并标记工单为“待补充信息”。

2. **初步诊断与优先级分配**
   - 动作：基于问题类型执行不同的诊断流程
   - **分支1：故障类**
     - 动作：查询 Confluence 知识库，按产品线和版本号过滤，寻找已知解决方案
     - 规则：若 30 分钟内未找到方案，自动升级至二线工程师团队
     - **规则**：升级二线时，必须附带完整对话记录、已尝试步骤、环境信息（如系统版本、日志片段）
     - 规则：若为严重故障（如影响全部用户或核心功能瘫痪），需同步通知客户经理和客户成功团队
     - **规则**：若发现与现有 open 工单问题重复，自动合并到母单，保留所有时间线 [待补充：合并触发条件细节，如基于标题关键词匹配或客户邮箱匹配]
   - **分支2：咨询类**
     - 动作：匹配 FAQ 知识库（按产品线和版本号过滤）
     - 规则：若匹配度低于 80%，转接至人工客服处理
   - **分支3：功能请求类**
     - 动作：记录客户需求，并转交产品团队
     - 规则：此类工单不计入故障 SLA 统计
     - [待补充：功能请求转交后的跟踪规则，如定期同步进展给客户]

3. **解决方案执行与回复**
   - 动作：提供解决方案或临时规避措施
   - 动作：通过邮件/Zendesk回复客户
   - **规则**：回复时引用工单号（INC-开头），若关联Jira缺陷单则一并注明
   - **规则**：若客户情绪激动（如使用抱怨、指责性语言），先发送安抚信息暂缓深入技术排查，待情绪稳定后再继续
   - **规则**：涉及数据安全或合规的问题走专门安全通道处理，禁止口头承诺修复时间
   - **规则**：周末只有值班工程师，仅 P1 级别工单可电话联系客户
   - **规则**：多语言工单优先使用客户使用的语言进行回复
   - **规则**：与研发协作用 Jira 创建 bug 并关联原 Zendesk 工单号

4. **工单备注与归档**
   - 动作：处理过程中及结单前，按固定格式填写工单备注
   - **规则**：备注格式为“问题摘要 / 处理步骤 / 结论 / 待办”，各部分用“/”分隔
   - **规则**：若存在重复工单，合并到母单后，子单备注中注明“已合并至母单 INC-xxxxx”
