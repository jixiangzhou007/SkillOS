# 技能名称：技术支持工单处理

## 触发条件
- 当客户通过邮件或 Zendesk 工单系统提交问题时触发。

## 处理流程

### 第一步：工单接收与初步分类
- **输入**：客户提交的原始工单（标题、描述、附件）。
- **动作**：
    1. 解析工单标题和描述，提取关键问题关键词（例如：“无法登录”、“报错”、“License过期”）。
    2. 根据关键词进行初步分类，分类包括：
        - `category_account`：权限/账号问题（如：密码错误、账号锁定）。
        - `category_bug`：系统Bug或配置错误（如：功能异常、版本不匹配）。
        - `category_pending`：信息不足，需要客户提供更多日志或截图。
        - `type_incident`：故障类，系统无法正常使用或出现错误（如：无法登录、页面报错、服务中断）。
        - `type_inquiry`：咨询类，客户需要操作指导或了解产品信息（如：如何导出报表、功能说明）。
        - `type_feature`：功能请求类，客户希望增加新功能或优化现有功能（如：希望增加批量导入功能）。
- **输出**：初步分类标签（包含原有的 `category_*` 和新识别的 `type_*`）及关联的风险等级（[待补充]）。

### 第二步：问题诊断与排查
- **条件**：基于初步分类结果，执行不同的诊断路径。
- **动作**：
    1. 如果是 `category_pending`，自动生成回复模板，向客户索取指定的日志文件或系统截图。
    2. 如果是 `category_account`，检查客户账号状态（如：是否过期、是否被锁定）。
    3. 如果是 `category_bug`，比对客户系统版本与已知Bug列表或配置规范。
    4. 如果是 `type_incident`，优先检查系统服务状态、网络连通性及客户本地环境（如：浏览器缓存、插件冲突）。**同时启动30分钟升级倒计时，若超时未找到方案则升级至二线工程师。**
    5. 如果是 `type_inquiry`，直接从知识库搜索匹配的操作指南或常见问题解答（FAQ）。**若匹配度低于80%，则将工单转人工客服处理。**
    6. 如果是 `type_feature`，记录客户需求描述、期望场景及业务价值，**转入产品团队需求池，不计入故障 SLA**。
- **输出**：诊断结果（包含解决方案或升级/转派记录）。

### 第三步：SLA 管理与响应
- **动作**：根据工单的优先级和分类，分配对应的 SLA 响应时间。
    - **P1（紧急故障）**：首次响应时间 ≤ 4小时。
    - **P2（一般故障/咨询）**：首次响应时间 ≤ 当天（24小时内）。
    - **P3（功能请求/非紧急咨询）**：首次响应时间 ≤ 3个工作日。
