# 技能名称：订单数据提取测试流程（Test Extraction Flow）

## 触发场景
- 当用户提交订单时触发
- 当系统接收到新的订单数据（如 JSON/XML 结构）时触发

## 前置条件
- [待补充] 需要确认订单数据源的格式（如 API 响应、消息队列消息）
- [待补充] 需要确认目标提取字段的基准值（如“商品名称”、“金额”、“支付渠道”）

## 执行步骤
1. **数据格式检查**：验证输入的订单数据结构是否符合预期（如 JSON 字段完整、类型正确）。
2. **字段映射确认**：确保测试环境中的字段名与生产环境一致，避免因环境差异导致提取失败。
3. **核心字段提取**：按定义好的规则（如正则、键值对）提取指定字段。
4. **结果比对**：将提取结果与基准值进行比对，判断是否一致。

## 输出/产出
- 提取结果报告（JSON 格式，包含提取值、比对结果、异常信息）
- 测试用例执行日志（Pass/Fail）

## 边界情况
- **空值或缺失字段**：如订单中缺少“金额”字段，需标记为异常并跳过。
- **格式错误**：如数据为纯文本而非 JSON，需提供明确的错误提示。
- **多值字段**：如“商品名称”为数组（如购买了多个商品），需明确提取策略（取第一个/全部）。
