# 采购审批

## 核心问题
标准化采购审批流程，确保预算可控、供应商合规、审批分级、验收闭环。

## When to use
- keywords: 采购申请, 审批流程, 采购审批, 采购流程, 采购单
- context: 用户在OA系统或对话中提出需要处理采购相关事宜时
- excludes: 用户询问采购政策、供应商名单或进行非流程性的采购咨询时，不要使用此技能

## Instructions
Follow these steps in order. Ask the user if anything is marked [待确认].

1.  **申请触发与信息采集**
    -   需求部门经办人在OA系统填写采购申请单。
    -   必须包含：采购物品/服务描述、预估金额、需求时间、推荐供应商（如有）、预算代码。
    -   输出：待审核的采购申请单。

2.  **预算与供应商资质核实**
    -   采购部专员接收申请后，并行开展以下核实：
        -   **预算核实**：确认对应成本中心是否有足够预算余额，且采购项目属于预算内项目。
        -   **供应商资质核实**：
            -   首次合作供应商：核查营业执照、经营范围、行业许可证、历史履约记录（如有）。
            -   已有合作供应商：核查信用评级、近期交付质量、是否存在违约风险。
    -   **风险判定与分支处理**：
        -   **预算不足** → 退回需求部门，建议调整预算或缩减采购范围。
        -   **资质不符（高风险）**：如经营范围完全不覆盖、存在重大法律风险或失信记录。 → 直接退回申请，并通知需求部门更换供应商或重新评估需求。 [待确认：退回后的重新提交流程及时效]
        -   **资质存疑（中风险）**：如经营范围未明确涵盖但实际业务能力可佐证、信用评级刚过及格线。 → 要求需求部门或供应商补充证明材料（如同类项目合同、资质说明函），材料齐全后进入正常审批流程。 [待确认：补充材料的SLA时限]
        -   **资质合格（低风险）**：资质齐全且信用良好。 → 直接进入下一流程。
    -   责任人：采购部专员。
    -   SLA时限：[待确认，建议24小时内完成核实]

3.  **金额分级审批**
    -   根据采购金额和步骤2的风险等级，执行以下审批路径：
        -   **金额 < 1万元**：部门负责人审批。
        -   **1万元 ≤ 金额 < 10万元**：部门负责人 + 采购总监审批。
        -   **10万元 ≤ 金额 < 50万元**：部门负责人 + 采购总监 + VP审批。
        -   **金额 ≥ 50万元**：部门负责人 + 采购总监 + VP + CEO审批。
        -   **特殊规则**：若步骤2中识别出**中风险**，即使金额未达上限，也需跳级至更高层级审批（如增加采购总监审批）。若为**高风险**，则流程终止。
    -   责任人：各级审批人。
    -   输出：审批通过/驳回/要求补充材料。

4.  **下单执行**
    -   审批通过后，采购部执行专员根据申请单生成采购订单（PO），发送至供应商确认，并跟踪订单状态。
    -   责任人：采购部执行专员。
    -   输出：已确认的PO、预计到货时间。

5.  **到货验收与异常处理**
    -   需求部门收到货物/服务后，进行验收（核对数量、质量、规格）。
    -   **验收合格**：完成验收单，流程归档。
    -   **验收不合格**：退回供应商处理，同时触发合同违约条款。
    -   责任人：需求部门验收人。
    -   输出：验收合格单/退货单及违约处理记录。

## Decision routes
| 用户意图/条件 | 执行动作 | 备注 |
|------------|---------|------|
| 用户说“提交采购申请”或“走采购流程” | 执行步骤1，引导用户填写采购申请单 | 确保信息完整 |
| 预算核实不足 | 执行步骤2中的预算不足分支，退回并建议调整 | 流程终止或重新提交 |
| 供应商资质不符（高风险） | 执行步骤2中的高风险分支，直接退回申请 | 流程终止 |
| 供应商资质存疑（中风险） | 执行步骤2中的中风险分支，要求补充材料 | 材料齐全后继续 |
| 审批通过 | 执行步骤4，生成并发送PO | 跟踪订单状态 |
| 验收不合格 | 执行步骤5中的不合格分支，退回并触发违约条款 | 流程闭环 |

## Inputs
- 采购清单: string, 必填，描述采购的物品或服务
- 预算代码: string, 必填，用于预算核实的成本中心代码
- 供应商信息: string, 必填，供应商名称及资质文件（如营业执照）
- 预估金额: number, 必填，采购的预估总金额
- 需求时间: date, 必填，期望的到货或完成时间

## Outputs
- status: enum — 执行结果（success / pending / rejected / escalated）
- message: string — 给用户或下游系统的摘要说明
- audit_notes: string — 关键操作记录（可选）
