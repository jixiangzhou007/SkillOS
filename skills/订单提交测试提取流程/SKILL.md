# 技能名称：订单提交测试提取流程
## 元数据
- 目标：test extraction flow
- 触发场景：用户提交订单时
- 版本：1.3

## 流程步骤
1. 接收输入数据
   - [待补充] 数据格式要求（如JSON/CSV）
   - [待补充] 字段映射规则（如订单号、商品名、数量）
   - [待补充] 异常字段处理策略（如复合字段、缺失字段）

2. 执行提取逻辑
   - [待补充] 提取规则集（如正则、关键词匹配）
   - [待补充] 优先级排序（如精确匹配 > 模糊匹配）
   - [新增] 复合字段拆分策略（如用户选择了“逗号+星号”拆分，则先按逗号分割子订单，再按星号分割商品名和数量）

3. 格式化输出结果
   - [待补充] 输出格式（如结构化表格、告警日志）
   - [待补充] 输出目标（如数据库、消息队列、测试报告）

## 边界情况
- 复合字段：如“蓝色L码运动鞋*2, 白色M码T恤*3”，需定义拆分策略（如用户选择“逗号+星号”拆分）
- 缺失字段：如缺少商品名，需定义默认值或报错机制
- 格式异常：如字段包含特殊符号，需定义清洗规则
- [新增] 复合字段拆分顺序：用户选择了“先逗号后星号”策略，后续需验证该顺序是否适用于所有场景
- **[新增] 验证冲突：** 订单ID格式正确，但订单所属客户ID与当前登录用户ID不匹配。需定义处理策略（如：高风险终止/中风险标记/低风险忽略）

## 前置条件
- [待补充] 用户已登录并提交有效订单
- [待补充] 订单数据校验通过（如必填项不为空）

## 输出/产出
- [待补充] 提取结果数据集（含成功/失败标识）
- [待补充] 异常日志（含错误码、字段位置、建议操作）

## 参考模板
- **[新增] [待补充]** 需要用户提供或定义一份订单/退款数据的参考模板（如JSON Schema或CSV列定义），以确保流程的输入输出格式统一。
