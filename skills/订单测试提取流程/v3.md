# 技能名称：订单测试提取流程

## MetaSkill 流水线
本流程由以下环节顺序执行：触发 → 输入校验 → 执行提取 → 输出 → 边界处理。

## 触发场景
- 当用户在前端提交订单并支付成功后，自动触发本流程。

## 输入/前置条件
- 订单数据必须包含以下核心字段：订单ID、客户ID、商品ID、商品SKU。
- 仅处理“已支付”状态的订单。

## 执行步骤
1. **验证订单**：校验订单数据完整性（包括订单ID、客户ID、商品ID、商品SKU是否非空）。若字段缺失，根据既定策略处理（如停止并告警、跳过、使用默认值）。

3. **计算金额**：[待补充] 计算逻辑是什么（如：按商品单价×数量，还是按订单总金额）？
4. **退款**：[待补充] 退款接口调用方式及重试机制。

## 边界情况
- **数据不完整**：若订单数据中商品SKU缺失，流程将依据预设策略处理（停止/跳过/默认值）。
