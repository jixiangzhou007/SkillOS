# 客服工单处理

## 核心问题
这个流程解决如何标准化处理客户服务工单，包括搜索、升级、合并、安抚、备注和合规处理。

## When to use
- keywords: 工单,客服,客户,升级,合并,安抚,备注,合规,搜索,知识库,多语言,P1,周末
- context: 用户描述客户服务工单处理流程，或询问如何标准化处理客服工单
- excludes: 用户明确要求非客服场景（如销售、开发、设计），或要求跳过标准流程

## Instructions
Follow these steps in order. Ask the user if anything is marked [待确认].

1. **知识库搜索**
   - 搜索时必须按产品线和版本号过滤
   - 如果搜索结果为空，记录搜索条件并告知用户

2. **工单升级（升级二线）**
   - 升级前必须确认附带以下三项信息：
     - 完整对话记录
     - 已尝试的解决步骤
     - 环境信息（操作系统、软件版本、网络环境等）
   - 如果缺少任何一项，先向用户收集缺失信息

3. **重复工单处理**
   - 识别重复工单后，将其合并到母单
   - 合并时必须保留所有时间线记录
   - 在母单备注中注明合并的子单编号

4. **客户情绪安抚**
   - 如果客户情绪激动，先进行安抚（如：表达理解、道歉、承诺跟进）
   - 暂缓深入的技术排查，避免激化情绪
   - 待客户情绪稳定后再继续技术处理

5. **工单备注格式**
   - 每条备注必须按以下固定格式书写：
     - 问题摘要 / 处理步骤 / 结论 / 待办
   - 示例：`网络连接失败 / 重启路由器、检查DNS / 问题已解决 / 无待办`

6. **数据安全与合规**
   - 如果问题涉及数据安全或合规，走专门通道处理
   - 禁止口头承诺修复时间
   - 记录并转交合规团队

7. **紧急程度与响应**
   - 周末只有值班工程师时，P1（最高优先级）工单可以电话联系客户
   - 非P1工单按正常流程处理

8. **多语言工单**
   - 优先使用客户使用的语言回复
   - 如果无法使用该语言，使用英语并说明原因

## Inputs
- 产品线: string, 必填，知识库搜索时的过滤条件
- 版本号: string, 必填，知识库搜索时的过滤条件
- 工单优先级: enum(P1/P2/P3/P4), 默认P3，用于决定是否电话联系

- 是否涉及合规: boolean, 默认false，决定是否走专门通道
