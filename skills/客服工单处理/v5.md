# 客服工单处理

## 核心问题
标准化处理客户技术支持工单，确保按产品线/版本号搜索知识库、规范升级二线、安抚情绪客户、合并重复工单、合规处理数据安全问题。

## When to use
- keywords: 工单, 客服, 技术支持, 客户问题, 升级, 二线, 安抚, 投诉, 重复工单, 合并, 知识库, 搜索, 产品线, 版本号, 数据安全, 合规, 周末, 值班, P1, 多语言, 英文, 日文
- context: 处理客户技术支持工单、升级二线、合并重复工单、安抚情绪客户、搜索知识库解决方案
- excludes: 非客服场景、非工单处理场景、仅需简单回复无需流程的场景

## Instructions
Follow these steps in order. Ask the user if anything is marked [待确认].

1. **识别工单类型与优先级**
   - 判断是否为重复工单：如果是，合并到母单，保留所有时间线记录
   - 判断客户情绪：如果情绪激动，先执行安抚步骤（第5步），暂缓深入技术排查
   - 判断是否涉及数据安全或合规：如果是，走专门通道处理，禁止口头承诺修复时间
   - 判断是否为周末值班：如果是且为P1优先级，可以电话联系客户

2. **知识库搜索**
   - 按产品线过滤搜索结果
   - 按版本号进一步过滤搜索结果
   - 如果找到匹配方案，直接回复客户

3. **工单备注记录**
   - 固定格式：`问题摘要 / 处理步骤 / 结论 / 待办`
   - 每个字段用斜杠分隔，内容简洁明确

4. **升级二线处理**
   - 升级前必须收集以下三项材料：
     a. 完整对话记录
     b. 已尝试的排查步骤
     c. 环境信息（操作系统、版本、配置等）
   - 如果缺少任何一项，先向客户索取再升级

5. **客户情绪安抚流程**
   - 如果客户情绪激动：
     a. 先表达理解和同理心
     b. 确认问题已记录，正在处理
     c. 暂缓深入技术排查或追问细节
     d. 待情绪稳定后再继续技术排查
   - 如果客户使用非中文语言：
     a. 优先使用客户使用的语言回复
     b. 确保翻译准确，避免歧义

6. **合规与安全处理**
   - 如果涉及数据安全或合规问题：
     a. 走专门通道处理
     b. 禁止口头承诺修复时间
     c. 记录工单后转交对应团队

## Decision routes
| 用户意图/条件 | 执行动作 | 备注 |
|------------|---------|------|
| 客户情绪激动 | 先安抚，暂缓技术排查 | 待情绪稳定后再继续 |
| 需要升级二线 | 收集完整对话、已尝试步骤、环境信息后升级 | 缺一不可 |
| 重复工单 | 合并到母单，保留所有时间线 | 不新建独立工单 |
| 涉及数据安全/合规 | 走专门通道，禁止口头承诺修复时间 | 转交对应团队 |
| 周末值班且为P1 | 可以电话联系客户 | 非P1按常规流程 |
| 多语言工单 | 优先使用客户语言回复 | 确保翻译准确 |
| 知识库搜索 | 按产品线+版本号过滤 | 精确匹配 |

## Inputs
- 工单ID: string, 必填, 当前处理的工单编号
- 客户语言: string, 默认"中文", 客户使用的语言
- 优先级: enum(P1/P2/P3/P4), 默认"P3", 工单优先级
