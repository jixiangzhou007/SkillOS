# 客服工单处理

## 核心问题
按照SLA和流程规范，高效处理客户提交的技术支持工单，确保问题闭环。

## When to use
- keywords: 工单, 故障, 支持, 问题, 报修, 需求, INC-, SLA, 升级, 关单
- context: 当用户描述一个需要技术支持或需要记录为工单的问题时
- excludes: 当用户只是询问产品功能或价格，不需要创建正式工单时

## Instructions
Follow these steps in order. Ask the user if anything is marked [待确认].

1.  **接收工单并分类**
    -   判断工单类型：功能请求 或 故障/咨询。
    -   如果是**功能请求**，记录需求并转交产品团队，不计入故障SLA。流程结束。
    -   如果是**故障/咨询**，继续执行第2步。

2.  **判断优先级并启动SLA计时**
    -   根据影响范围确定优先级：
        -   P1（严重故障/生产不可用）：4小时内首次响应。
        -   P2（一般故障/功能受阻）：当天（24小时内）首次响应。
        -   P3（轻微问题/咨询）：3个工作日内首次响应。
    -   记录工单编号，格式为 `INC-` 开头（例如 `INC-20231027-001`）。

3.  **信息收集与验证**
    -   检查工单中是否包含以下必要信息：
        -   问题描述
        -   日志/截图
        -   产品版本号
        -   环境信息（操作系统、浏览器等）
    -   **如果信息不全**：使用模板邮件向客户索取缺失信息（日志、截图、产品版本号）。等待客户回复，工单状态保持 `等待客户`。
    -   **如果信息齐全**：继续执行第4步。

4.  **尝试一线解决**
    -   基于已有信息，尝试标准解决方案或知识库搜索。
    -   **搜索知识库时**：必须按客户提供的**产品线**和**版本号**进行过滤，以提高准确性。
    -   如果问题解决，跳至第6步（关单确认）。
    -   如果问题无法解决，继续执行第5步。

5.  **升级至二线支持**
    -   在升级工单前，必须确保附带以下完整信息包：
        -   完整对话记录
        -   已尝试的解决步骤
        -   环境信息
    -   将工单指派给二线支持团队。

6.  **关单确认（关键步骤）**
    -   在关闭工单前，**必须**获得客户确认问题已解决。
    -   如果客户确认，关闭工单（状态设为 `已解决/关闭`）。
    -   如果客户未确认或问题未解决，工单保持 `处理中` 状态，返回第4步或第5步。

7.  **严重故障特殊通知**
    -   如果工单被判定为P1严重故障，在启动SLA的同时，必须同步通知：
        -   客户经理
        -   客户成功团队

## Decision routes
| 用户意图/条件 | 执行动作 | 备注 |
|------------|---------|------|
| 用户提交功能请求 | 记录需求并转产品团队，不计入SLA | 流程结束 |
| 用户提交故障，信息齐全 | 尝试一线解决或升级二线 | 按SLA计时 |
| 用户提交故障，信息不全 | 发送模板邮件索取日志、截图、版本号 | 工单状态置为等待客户 |
| 问题已解决，需关单 | 必须获得客户确认 | 否则保持open |
| 故障为P1严重级别 | 同步通知客户经理和客户成功团队 | 同时启动4小时SLA |
| 需要升级二线 | 附带完整对话、已尝试步骤、环境信息 | 信息不全禁止升级 |
