# 客服工单处理

## 核心问题
按照SLA和流程规范，高效处理客户提交的技术支持工单，确保问题闭环。

## When to use
- keywords: 工单, 故障, 支持, 问题, 报修, 需求, INC-, SLA, 升级, 关单
- context: 当用户描述一个需要技术支持或需要记录为工单的问题时
- excludes: 当用户只是询问产品功能或价格，不需要创建正式工单时

## Instructions
Follow these steps in order. Ask the user if anything is marked [待确认].

1.  **接收工单并分类**
    -   判断工单类型：功能请求 或 故障/咨询。
    -   如果是**功能请求**，记录需求并转交产品团队，不计入故障SLA。流程结束。
    -   如果是**故障/咨询**，继续执行第2步。

2.  **判断优先级并启动SLA计时**
    -   根据影响范围确定优先级：
        -   P1（严重故障/生产不可用）：4小时内首次响应。
        -   P2（一般故障/功能受阻）：当天（24小时内）首次响应。
        -   P3（轻微问题/咨询）：3个工作日内首次响应。
    -   记录工单编号，格式为 `INC-` 开头（例如 `INC-20231027-001`）。

3.  **信息收集与验证**
    -   检查工单中是否包含以下必要信息：
        -   问题描述
        -   日志/截图
        -   产品版本号
        -   环境信息（操作系统、浏览器等）
    -   **如果信息不全**：使用模板邮件向客户索取缺失信息（日志、截图、产品版本号）。等待客户回复，工单状态保持 `等待客户`。
    -   **如果信息齐全**：继续执行第4步。

4.  **重复工单识别与合并** [新增]
    -   在开始处理前，检查当前工单是否与系统中其他 `open` 或 `in-progress` 的工单描述、客户、产品高度相似。
    -   **如果发现重复**：
        -   确定**母单**（通常为创建时间最早的工单）。
        -   将当前工单标记为 **“重复”** ，并关联到母单。
        -   将当前工单的所有沟通记录、附件、时间线**合并**到母单中。
        -   将当前工单状态更新为 **“关闭-重复”** ，并通知客户已在母单 `INC-xxxx` 中处理。
        -   后续所有处理工作仅在母单上进行。
    -   **如果无重复**：继续执行第5步。

5.  **尝试一线解决**
    -   基于已有信息，尝试标准解决方案或知识库搜索。
    -   **搜索知识库时**：必须按客户提供的**产品线**和**版本号**进行过滤，以提高准确性。
    -   如果问题解决，跳至第7步（关单确认）。
    -   如果问题无法解决，继续执行第6步。

6.  **升级二线** [新增]
    -   当一线无法解决时，升级到二线技术支持。
    -   **升级前必须附带**：
        -   完整的工单对话记录。
        -   已尝试过的所有解决步骤及结果。
        -   客户的环境信息（操作系统、浏览器、日志、截图等）。
    -   升级后将工单状态更新为 `已升级`。

7.  **关单确认** [修订]
    -   在关闭工单前，**必须获得客户确认问题已解决**。
    -   如果客户未确认，工单状态保持 `open` 或 `等待客户反馈`。
    -   **对于严重故障（P1）**：在关单前，必须**同步通知客户经理和客户成功团队**，确保他们知晓。
    -   获得确认后，更新工单状态为 `已关闭`。
