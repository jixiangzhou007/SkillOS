# 客服工单处理

## 核心问题
标准化处理客户技术支持工单，确保按产品线/版本号搜索知识库、规范升级二线、安抚情绪客户、合并重复工单、合规处理数据安全问题、规范与研发协作创建Bug并关联工单。

- excludes: 非客服场景、非工单处理场景、仅需简单回复无需流程的场景

## Instructions
Follow these steps in order. Ask the user if anything is marked [待确认].

1. **识别工单类型与优先级**
   - 判断是否为重复工单：如果是，合并到母单，保留所有时间线记录
   - 判断客户情绪：如果情绪激动，先执行安抚步骤（第5步），暂缓深入技术排查
   - 判断是否涉及数据安全或合规：如果是，走专门通道处理，禁止口头承诺修复时间
   - 判断是否为周末值班：如果是且为P1优先级，可以电话联系客户

2. **知识库搜索**
   - 按产品线过滤搜索结果
   - 按版本号进一步过滤搜索结果
   - 如果找到匹配方案，直接回复客户

3. **工单备注记录**
   - 固定格式：`问题摘要 / 处理步骤 / 结论 / 待办`
   - 每个字段用斜杠分隔，内容简洁明确

4. **升级二线处理**
   - 升级前必须收集以下三项材料：
     a. 完整对话记录
     b. 已尝试的排查步骤
     c. 环境信息（操作系统、版本、配置等）
   - 如果缺少任何一项，先向客户索取再升级

   - 触发条件：[待补充]（例如：确定为产品代码缺陷、无法通过知识库或二线解决）
   - 前置操作：[待补充]（例如：先在工单备注中按固定格式记录完整排查信息）
   - 创建动作：在 Jira 中创建 Bug，并在 Bug 描述或关联字段中填入原 Zendesk 工单号
   - 关联动作：在原 Zendesk 工单中备注或使用插件关联对应的 Jira Bug ID，确保双向可追溯
   - 沟通规范：[待补充]（例如：是否需同步告知客户已转研发、预计响应时间）

6. **客户情绪安抚流程**
   - 如果客户情绪激动：
     a. 先表达理解和同理心
     b. 确认问题已记录，正在处理
     c. 暂缓深入技术排查或追问细节
     d. 待情绪稳定后再继续技术排查
   - 如果客户使用非中文语言：
     a. 优先使用客户使用的语言回复
     b. 确保翻译准确，避免歧义
