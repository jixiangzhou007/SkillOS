# 严重故障处理

## 核心问题
规范严重故障的处理流程，确保客户经理和客户成功团队同步收到通知，工单按标准格式管理，升级和合并操作有据可依。

## When to use
- keywords: 严重故障, 故障通知, 工单升级, 重复工单, 客户安抚, 数据安全, 合规, INC-, Jira 关联, 知识库搜索
- context: 用户报告严重故障、要求升级二线、发现重复工单、或涉及数据安全/合规问题时使用
- excludes: 普通咨询、非故障类工单、已关闭工单的后续讨论

## Instructions
Follow these steps in order. Ask the user if anything is marked [待确认].

1. 确认工单是否为严重故障（如服务中断、数据丢失、安全事件等）。
   - 如果是严重故障，执行步骤 2。
   - 如果不是，按普通工单流程处理。
2. 同步通知客户经理和客户成功团队。
   - 通知方式：[待确认，如邮件、即时消息、系统通知]
   - 通知内容：工单编号、故障概述、影响范围、当前状态。
3. 检查工单号格式是否为 `INC-` 开头。
   - 如果格式正确，继续。
   - 如果格式不正确，修正为 `INC-` 开头。
4. 检查是否与 Jira 缺陷单关联。
   - 如果有关联，建立双向链接（工单→Jira，Jira→工单）。
   - 如果无关联，标记为待关联。
5. 搜索知识库，按产品线和版本号过滤，查找已知解决方案。
   - 如果找到匹配方案，记录到工单备注中，并尝试应用。
   - 如果未找到，标记为待二线处理。
6. 判断是否需要升级到二线支持。
   - 如果需要升级，执行步骤 7。
   - 如果不需要，继续当前处理。
7. 升级二线时，必须附带以下信息：
   - 完整对话记录
   - 已尝试的解决步骤
   - 环境信息（操作系统、版本、配置等）
8. 检查是否存在重复工单。
   - 如果存在重复工单，合并到母单（保留所有时间线）。
   - 如果不存在，继续。
9. 按固定格式填写工单备注：`问题摘要 / 处理步骤 / 结论 / 待办`
   - 问题摘要：简要描述故障现象
   - 处理步骤：已执行的操作列表
   - 结论：当前判断或结果
   - 待办：下一步需要完成的事项
10. 评估客户情绪状态。
    - 如果客户情绪激动，先安抚客户，暂缓深入技术排查。
    - 如果客户情绪平稳，继续技术排查。
11. 检查是否涉及数据安全或合规问题。
    - 如果是，走专门通道处理，禁止口头承诺修复时间。
    - 如果不是，按常规流程处理。
