## 核心问题
为50人创业公司设计一套飞书报销审批的自动化处理流程，确保财务合规、审批高效、异常可处理。

## S_body

### 核心步骤（按执行者最容易理解的顺序）

1. **材料完整性检查** → 附件必须包含发票PDF和付款截图
   - 为什么：确保报销有据可查，防止虚假报销
   - 失败处理：驳回报销单，通知员工补充

2. **基本信息核对** → 部门、费用类型（差旅/办公/招待）、发票抬头（公司全称）
   - 为什么：确保报销归属清晰，符合财务规范
   - 失败处理：逐项驳回要求修正

3. **重复报销检测** → 检查发票号码是否已存在历史记录
   - 为什么：防止同一发票多次报销，避免财务损失
   - 重复处理：系统自动拦截，通知财务和申请人

4. **金额分级审批** → 按金额决定审批人
   - ≤500元：部门负责人审批
   - 500-5000元：CFO审批
   - >5000元：CEO审批+加签确认
   - 为什么：金额越大风险越高，需要更高层级把关

5. **特殊费用类型附加检查** → 差旅费需行程单，招待费需客户名称
   - 为什么：确保费用真实性和业务关联性
   - 失败处理：驳回要求补充

6. **最终输出** → 生成付款台账Excel → 同步用友云 → 飞书群发已打款提醒
   - 为什么：完成财务闭环，确保数据一致性和透明度

## S_trigger

**触发条件**：员工在飞书提交「费用报销」模板报销单

**不应触发**：
- 预付款/借款申请（走其他流程）
- 非报销类财务审批
