# 技能名称：电商客服退款标准流程（淘宝数码配件小店版）
## 核心问题
规范、高效地处理淘宝小店（日单200+，主营数码配件）的各类退款请求，平衡客户体验与商家利益，减少纠纷与损失。
## 执行步骤（草稿）
1.  **受理与核实（订单层）**：接收退款申请后，立即核对订单信息。
    -   **关键核查点**：订单号、商品信息、购买账号、支付金额、物流单号、**签收时间**（与系统记录的7天/15天无理由退货期比对）。
    -   **材料核查**：初步审核顾客上传的凭证（如照片、视频、检测报告）是否清晰、完整、有效。
    -   **新增：物流状态交叉验证**：优先查看物流轨迹的**最后一条更新**（如“已签收”、“派送失败”、“退回”），并与顾客的**签收时间**及**包装照片**进行比对。如果物流显示“已由门卫/代收点代签”，需特别留意包装破损的责任归属。
2.  **责任判定（争议层）**：根据核实结果，判断退款责任方。
    -   **类型**：7天无理由、质量问题（商家责任）、物流问题、顾客主观原因（不想要、买错等）、恶意退款。
    -   **边界情况**：签收时间争议（如本例）、凭证模糊、超出退换货周期、缺少原始包装。
    -   **新增：物流损坏判定**：若照片显示外包装有明显挤压、水渍，且物流状态为“已签收”（非本人），优先引导顾客**拒收**或联系物流索赔。若顾客已签收，则需判断其是否已拆封检查。
    -   **新增：签收与证据冲突处理**：当物流记录（显示本人签收）与顾客提供的照片（显示外包装破损）冲突时，**[待补充：具体判定优先级和操作流程]**。应优先要求顾客提供**完整的开箱视频**或**内部损坏特写**作为核心证据，同时联系快递公司调取**签收底单**或**派送员通话录音**，核实签收时是否确认外观完好。
3.  **方案制定（策略层）**：根据责任判定结果，给出具体处理方案。
    -   **全额退款**：适用于7天无理由、确认的质量问题。
    -   **部分退款**：适用于商品有瑕疵但顾客愿保留、退货邮费分摊。
    -   **拒绝退款**：适用于超出周期、恶意退款、无法提供有效凭证。
    -   **升级处理**：涉及大额、高价值（如本例的5000元笔记本）、疑似欺诈或复杂纠纷时，升级至主管或风控部门。
    -   **新增：物流争议方案**：若判定为物流责任，方案应为：① 指导顾客联系快递索赔；② 商家协助发起物流理赔；③ 若顾客不配合，可考虑**先行赔付**（小额），再向物流追偿。
    -   **新增：证据不足时的临时方案**：若顾客无法提供有效的开箱视频或内部损坏证据，且物流记录清晰，可**暂不退款**，改为引导顾客申请**退货退款**，并要求其将商品原样寄回，收到后由商家检测确认损坏原因后再处理。
4.  **执行退款（系统层）**：与顾客达成一致后，在客服系统/ERP中操作。
