# 技能名称：电商客服退款标准流程
## 核心问题
规范、高效地处理电商平台上的各类退款请求，平衡客户体验与商家利益，减少纠纷与损失。
## 执行步骤（草稿）
1.  **受理与核实（订单层）**：接收退款申请后，立即核对订单信息。
    -   **关键核查点**：订单号、商品信息、购买账号、支付金额、物流单号、**签收时间**（与系统记录的7天/15天无理由退货期比对）。
    -   **材料核查**：初步审核顾客上传的凭证（如照片、视频、检测报告）是否清晰、完整、有效。
2.  **责任判定（争议层）**：根据核实结果，判断退款责任方。
    -   **类型**：7天无理由、质量问题（商家责任）、物流问题、顾客主观原因（不想要、买错等）、恶意退款。
    -   **边界情况**：签收时间争议（如本例）、凭证模糊、超出退换货周期、缺少原始包装。
3.  **方案制定（策略层）**：根据责任判定结果，给出具体处理方案。
    -   **全额退款**：适用于7天无理由、确认的质量问题。
    -   **部分退款**：适用于商品有瑕疵但顾客愿保留、退货邮费分摊。
    -   **拒绝退款**：适用于超出周期、恶意退款、无法提供有效凭证。
    -   **升级处理**：涉及大额、高价值（如本例的5000元笔记本）、疑似欺诈或复杂纠纷时，升级至主管或风控部门。
4.  **执行退款（系统层）**：与顾客达成一致后，在客服系统/ERP中操作。
    -   **操作流**：确认退款金额 → 选择退款渠道（原路返回/钱包/其他）→ 提交审批（如需）→ 完成退款。
    -   **通知**：退款成功后，系统自动发送通知给顾客。
5.  **闭环记录（知识层）**：将本次退款案例的关键信息记录归档，用于优化规则和培训。
    -   **记录点**：退款原因、责任判定依据、处理方案、最终结果、顾客反馈。
    -   **用途**：生成客服FAQ、优化商品描述、识别高风险顾客/商品。
## 输入/前置条件
-   顾客发起的退款申请单（含订单号、退款原因、金额、凭证）。
-   订单后台数据（物流信息、支付信息、顾客历史记录）。
-   平台/店铺的退换货政策。
## 输出/产出
-   退款处理结果（成功/失败/部分成功）。
-   系统内的退款操作记录。
-   客服与顾客的沟通记录（含方案协商过程）。
-   归档的案例知识库条目。
