# 技能名称：合同审核 - 条款比对与风险识别
## Core Problem
在合同审核过程中，快速定位关键条款（如违约责任、管辖、付款条件）与前期约定（如备忘录、框架协议、招标文件）之间的差异，并识别风险等级。

## S_route
| Intent | Action | Resource |
| :--- | :--- | :--- |
| 接收待审合同与参考文件 | 1. 获取合同电子版（Word/PDF）<br>2. 获取所有相关前期文件（邮件、备忘录、招标书、报价单）<br>3. 确认审核范围（是全面审还是只审修改点） | 文件管理系统 / 邮件归档 |
| 条款比对（核心步骤） | 1. 使用工具（如对比软件）或人工逐条比对关键条款<br>2. 重点关注：违约金比例、争议解决地、付款节点、免责范围<br>3. 标记出与前期文件不一致的条款 | [待补充：具体比对工具] |
| 风险等级评定 | 1. 对每个差异点按“法律后果严重性”和“对方是否可能接受修改”打分<br>2. 输出：高风险（必须改）、中风险（建议改）、低风险（仅备注） | 风险评级矩阵 |

## 边界情况处理
- **无前期文件**：若没有备忘录或邮件记录，则默认以合同正文为准，仅对显失公平的条款（如违约金过高）进行提示。
- **口头约定**：除非有录音或第三方证明，否则口头约定不作为修改依据，仅在备注中提醒“建议补充书面确认”。
- **模板合同**：如果是对方提供的标准模板（如采购、租赁），需要额外关注格式条款（如“最终解释权归甲方”）。
