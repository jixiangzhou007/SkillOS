# 工单处理与升级

## 核心问题
标准化客服工单处理流程：合并重复工单、安抚情绪客户、规范备注、安全合规升级、周末值班响应、多语言回复。

## When to use
- keywords: 工单, 合并, 重复, 情绪, 安抚, 升级, 安全, 合规, Jira, Zendesk, 周末, 值班, P1, 多语言, 备注
- context: 客服人员处理客户工单，需要执行合并、安抚、升级、跨部门协作或特殊场景（周末/多语言）响应时。
- excludes: 工单不涉及客户投诉或内部协作（如：纯信息查询）；或用户明确要求不遵循标准流程。

## Instructions
Follow these steps in order. Ask the user if anything is marked [待确认].

1.  **识别重复工单**：检查当前工单是否与已有工单描述同一问题。
    -   IF 是重复工单，THEN 将其合并到母单（原始工单）。
    -   合并时，**保留所有时间线**（包括子单的创建、更新、备注时间）。
2.  **评估客户情绪**：判断客户是否情绪激动。
    -   IF 客户情绪激动，THEN 执行安抚步骤（如：表达理解、道歉、承诺跟进），**暂缓深入的技术排查**，避免激化矛盾。
3.  **规范工单备注**：所有内部备注和对外回复必须包含以下四个部分，格式固定：
    -   **问题摘要**：一句话概括客户问题。
    -   **处理步骤**：已执行的操作列表。
    -   **结论**：当前状态或最终结论。
    -   **待办**：下一步需要执行的动作。
4.  **判断安全合规风险**：评估工单内容是否涉及数据安全、隐私或合规问题。
    -   IF 涉及，THEN **走专门的安全/合规通道**（如：提交内部安全工单、通知安全团队），**禁止口头承诺客户修复时间**。
5.  **判断是否需要研发协作**：评估问题是否需要修改代码或配置。
    -   IF 需要研发协作，THEN 在 Jira 中创建 Bug 任务，并**关联原 Zendesk 工单**。
6.  **判断响应时间与渠道**：
    -   IF 当前是周末且只有值班工程师，THEN 仅对 P1（最高优先级）工单通过电话联系客户。
    -   IF 工单语言非中文/英文，THEN 优先使用客户所使用的语言进行回复。
7.  **执行升级或回复**：根据以上判断结果，执行对应的操作（合并、安抚、备注、创建Jira、升级安全通道、电话联系或语言回复）。

## Decision routes
| 用户意图/条件 | 执行动作 | 备注 |
|------------|---------|------|
| 客户重复提交工单 | 合并到母单，保留所有时间线 | 母单为最早创建的工单 |
| 客户情绪激动 | 先安抚，暂缓技术排查 | 避免在客户激动时讨论技术细节 |
| 工单涉及数据安全或合规 | 走专门安全/合规通道，禁止口头承诺修复时间 | 必须通过正式流程记录 |
| 需要研发修改代码/配置 | 在Jira创建Bug，关联原Zendesk工单 | 确保双向可追溯 |
| 周末值班，工单优先级为P1 | 电话联系客户 | 非P1工单 [待确认] 如何处理 |
| 客户使用非中文/英文 | 使用客户语言回复 | [待确认] 是否支持所有语言 |
| 需要记录工单处理过程 | 按格式备注：问题摘要/处理步骤/结论/待办 | 所有备注必须遵循此格式 |
