# 技能名称：客户退款处理（生鲜与虚拟商品增强版）
## Core Problem
处理客户退款申请，根据商品类型、签收时间、退款原因进行精准分流与决策。
## S_route
| Intent | Action | Resource |
| :--- | :--- | :--- |
| 接收退款请求 | 获取订单号、商品类型、签收时间、退款原因 | 客服工单系统 |
| 商品类型判断 | 判断商品为“普通商品”、“生鲜食品”或“虚拟商品” | 商品分类数据库 |
| 签收时效校验 | 计算当前时间与签收时间的差值（小时/天） | 订单物流系统 |
| 退款原因分类 | 将退款原因归类为“质量问题”、“主观感受”、“无理由”等 | 预设原因词库 |
| 策略匹配与结果输出 | 根据前三步结果，匹配预设的退款/换货/拒绝策略 | [待补充：策略规则引擎] |
## S_body
1.  **接收退款请求**：确认订单号、用户身份、购买商品及金额是否与退款申请一致。
2.  **商品类型判断**：
    - 若商品为“生鲜食品”，跳转至3.1。
    - 若商品为“虚拟商品”，跳转至3.2。
    - 若商品为“普通商品”，跳转至3.3。
3.  **时效与原因综合判断**：
    - **3.1 生鲜食品**：
        - 签收≤24小时 → 进入“可退款”队列。
        - 签收>24小时 → 拒绝退款，并建议用户检查商品状态或联系供应商。
        - **边界情况**：若退款原因为“商品腐烂/变质”，即使超24小时，也应升级为人工处理（转工单）。
    - **3.2 虚拟商品**：
        - 无论签收时间，若已发送/激活 → 拒绝退款，并告知用户“虚拟商品一经发送概不退换”。
        - 若未发送 → 可全额退款。
    - **3.3 普通商品**：
        - 签收≤7天 → 可退款。
        - 签收≤15天且>7天 → 仅支持换货。
        - 签收>15天 → 拒绝退款/换货。
