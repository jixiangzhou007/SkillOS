# 技能名称：销售合同审核（初级版）

## 触发条件
- 用户收到一份对方发来的销售合同文件（如PDF、Word）。

## 技能步骤
1.  **第一步：核对合同内容与价格**
    -   **动作**：读取合同中的“产品清单及价格”部分。
    -   **核对项**：
        -   **产品名称/型号**：是否与内部报价单或订单一致。
        -   **单价**：是否与内部报价单一致。
        -   **数量**：是否与订单一致。
        -   **总价**：单价×数量是否计算正确。
        -   **额外项目**：是否有合同中多出的、未经确认的收费项目（如技术服务费、运费、税费等）。
    -   **风险判定**：
        -   **高风险**：单价/数量错误、存在未经确认的额外收费项目 → 必须修改。
        -   **中风险**：总价计算错误（但单价和数量正确）→ 建议修改。
        -   **低风险**：格式或单位标注不清 → 仅标注。

2.  **第二步：识别关键法律风险条款**
    -   **动作**：逐条审查合同中的法律风险条款。
    -   **审查项**：
        -   **知识产权归属**：明确开发成果、背景知识产权（如核心代码库）的归属权。
            -   **风险判定**：
                -   **高风险**：约定我方背景知识产权（如核心代码、框架）归对方所有，或约定我方所有开发成果（含背景IP）无限制归属对方 → 必须修改。
                -   **中风险**：约定项目定制部分归对方所有，但未对我方背景IP进行明确保护或授权范围过宽 → 建议修改。
                -   **低风险**：约定清晰，我方背景IP保留，仅授权对方使用定制成果 → 仅标注。

        -   **对等违约金**：[待补充]
