# 技能名称：销售合同审核（初版）
## Core Problem
对销售合同文本进行系统性审核，重点识别价格、付款、产品描述等核心条款的风险，保护己方利益。

## S_route
| Intent | Action | Resource |
| :--- | :--- | :--- |
| 条款比对与核心利益审查 | 将待审合同与**标准模板/历史版本/谈判纪要**进行逐条比对，**优先审查“产品/服务描述”（含规格、数量、质保期）和“价格/付款条款”（含单价、总价、付款比例、节点、违约金）**，标记所有差异点。 | 合同标准模板、历史合同库、谈判记录、[待补充：行业标准条款库] |
| 风险识别与分级 | 对差异条款进行风险评估，按“高风险（必须改）”、“中风险（建议改）”、“低风险（仅标注）”三级分类。**其中，付款比例、质保期、核心交付标准等直接影响利润或法律责任的差异，默认至少为中风险。** | 风险分级标准表、法律法规库、[待补充：公司内部风险偏好] |
| 修改建议与谈判策略 | 针对中、高风险条款，输出修改措辞建议，并提供谈判底线与让步策略。 | 修改措辞库、谈判话术模板 |
## 新增关注要点
### 1. 知识产权归属审查
- **触发条件**：当合同中包含“知识产权”、“著作权”、“专利权”、“技术成果”等关键词时。
- **审查重点**：
  - 明确合作中产生的知识产权归属（哪一方所有？共有？）。
  - 审查是否有授权使用费、转让费、许可范围等限制条款。
  - 默认策略：**如对方试图将合作产生的知识产权完全归为己有，且我方贡献核心数据或算法，应标记为高风险，必须修改为双方共有或我方所有。**
### 2. 免责条款与责任限制
- **触发条件**：当合同中包含“免责”、“责任上限”、“不可抗力”、“间接损失”等关键词时。
- **审查重点**：
  - 免责范围是否过于宽泛（例如：免除因重大过失或故意违约的责任）。
  - 责任上限是否过低（例如：赔偿上限远低于合同总价或预期损失）。
  - 默认策略：**如对方将核心交付责任（如数据泄露、交付延迟导致的生产损失）也纳入免责或责任上限范围，应标记为高风险。**
### 3. 对等违约金条款
- **触发条件**：当合同中包含“违约金”、“罚金”、“赔偿金”等关键词时。
- **审查重点**：
  - 双方违约金比例是否对等（例如：我方逾期交货罚 1%/天，对方逾期付款罚 0.1%/天，则为不对等）。
  - 违约金上限是否合理（例如：上限为合同总价的 10% 以下，且未区分违约严重程度）。
  - 默认策略：**如违约金不对等（差距超过 2 倍）或上限过低（低于合同总价 10%），标记为高风险。**
