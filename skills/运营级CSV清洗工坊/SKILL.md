# 运营级CSV清洗工坊

## 核心问题
销售导出的CSV文件存在重复行、空邮箱、金额格式混乱、日期格式不统一等问题，运营同学需要一键清洗为干净数据并附带清洗报告。

## When to use
- keywords: CSV清洗, 数据清洗, 去重, 重复行, 空邮箱, 金额格式, 日期格式, 清洗报告, 销售导出, 运营数据
- context: 运营同学处理销售导出的CSV文件，需要清理重复数据、修复格式问题
- excludes: 用户上传的是图片、PDF、Word文档；用户需要的是数据分析或可视化而非清洗；用户需要的是实时数据同步而非文件清洗

## Instructions
Follow these steps in order. Ask the user if anything is marked [待确认].

1. **导入与预览**：读取用户提供的CSV文件，自动识别列名和数据类型，展示前5行数据摘要，让运营同学确认文件无误。如果文件无法读取（如编码问题），提示用户重新导出为UTF-8编码的CSV。
2. **按主键id去重**：
   - 查找名为`id`的列，如果不存在则提示用户指定主键列名。
   - 保留id最早出现的那一行，删除后续重复id的行。
   - 如果存在完全重复行（所有字段相同），仅保留第一条。
   - 如果存在关键字段重复（如邮箱+手机号完全一致但id不同），标记为“疑似重复”，在清洗报告中列出，由运营同学人工确认。
   - [待确认] 关键字段的默认组合（如邮箱+手机号）是否可由运营在配置中自定义。
3. **空邮箱与格式校验**：
   - 检查email列（或用户指定的邮箱列名），将email为空或格式不合法（如缺少@、域名无效）的行，**不静默删除**，单独导出到`anomaly_rows.csv`文件。
   - 主表中保留合法邮箱的行。
   - 在清洗报告中记录异常行数。
4. **金额格式标准化**：
   - 检查amount列（或用户指定的金额列名），将常见格式如`1,234.56`、`¥1234.56`、`-1234.56`统一转为纯数字格式（如`1234.56`）。
   - 移除货币符号（¥、$、€等）与千分位逗号。
   - 负数保留负号并标红（在报告中注明标红处理）。
   - 转成decimal类型。
5. **日期统一化**：
   - 检查date列（或用户指定的日期列名），将所有日期字段统一为`YYYY-MM-DD`格式。
   - 无法解析的日期值（如“2024/13/01”、“abc”）所在行，单独导出到`anomaly_rows.csv`，并在报告中记录。
6. **导出与报告**：
   - 生成清洗后的CSV文件（命名为`cleaned_<原文件名>.csv`）。
   - 自动输出一份TXT格式的清洗日志（命名为`cleaning_report_<原文件名>.txt`），包含：
     - 总行数
     - 去重数（完全重复行数 + 主键重复行数）
     - 异常数（空邮箱行数 + 格式非法邮箱行数 + 无法解析日期行数）
     - 各字段填充率（非空行数/总行数，按百分比显示）
   - 将清洗日志内容展示给用户，并告知输出文件位置。

## Decision routes
| 用户意图/条件 | 执行动作 | 备注 |
|------------|---------|------|
| 用户上传CSV文件，要求清洗 | 执行完整清洗流程（步骤1-6） | 默认流程 |
| 用户指定了不同的主键列名 | 使用用户指定的列名替代默认的`id`进行去重 | 需在步骤2前确认列名是否存在 |
| 用户指定了不同的金额列名或日期列名 | 使用用户指定的列名进行格式标准化 | 需在步骤4/5前确认列名是否存在 |
| 清洗过程中发现异常行 | 导出到`anomaly_rows.csv`，在报告中记录，不中断流程 | 异常行不丢失 |
| 用户上传的文件不是CSV格式 | 提示用户重新导出为CSV格式 | 非CSV文件不处理 |

## Inputs
- input_file: string, 无默认值, 用户上传的CSV文件路径
- primary_key: string, 默认"id", 用于去重的主键列名
- amount_column: string, 默认"amount", 需要执行金额格式标准化的列名
- date_column: string, 默认"date", 需要执行日期统一化的列名
- email_column: string, 默认"email", 需要校验邮箱格式的列名
- anomaly_output: string, 默认"anomaly_rows.csv", 异常行导出文件名
- output_prefix: string, 默认"cleaned_", 清洗后文件的前缀
