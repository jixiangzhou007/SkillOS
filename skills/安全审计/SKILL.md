# 安全审计

## 核心问题
标准化执行安全审计任务，确保审计范围明确、证据充分、整改闭环，最终合规结案。

## When to use
- keywords: 安全审计, 审计任务, 审计流程, 合规检查, 审计工单
- context: 用户收到审计任务（系统工单、邮件指令、口头指派），需要按标准流程执行安全审计
- excludes: 非安全相关的审计（如财务审计、代码审计）、日常安全检查（非正式审计任务）

## Instructions
Follow these steps in order. Ask the user if anything is marked [待确认].

1. **接收审计任务与初步沟通**
   - 记录审计发起方、审计对象（系统/部门/流程）、审计背景（如：合规检查、事件响应、例行抽查）。
   - 与业务方确认是否存在已知风险或近期变更。
   - 要求提供基础资料：系统架构图、权限清单、近期操作日志、历史审计报告、相关制度文件。
   - **SLA确认**：记录任务的紧急程度（紧急/常规）和完成时限（如：紧急任务24小时内出具初步报告）。

2. **确定审计范围与标准**
   - **范围划定**：明确审计边界（如：仅审计生产环境、是否包含第三方接口、时间跨度）。根据审计背景和初步沟通结果，决定是“聚焦特定事件”还是“扩大范围排查”。
   - **标准确定**：依据法律（《数据安全法》《个人信息保护法》等）、行业标准（ISO 27001、等保2.0）、公司内部制度，确定审计的“合规基线”。
   - **冲突处理**：若内部制度与法律冲突，以法律为准；若标准缺失，标记为“[待补充]”并建议修订制度。同时，在本次审计中，需与审计对象确认是否存在可参考的“惯例”或“临时流程”，作为本次审计的补充判断依据。
   - **审批链**：审计范围与标准的确定，需经过审计负责人审批确认。

3. **收集证据与穿行测试**
   - **证据收集**：调取系统日志、审批记录、配置文档、访谈记录。
   - **穿行测试**：选取1-2条典型业务流，从头到尾模拟操作，验证控制点是否有效。
   - **异常标记**：对无审批、无日志、越权操作等行为，截图/录屏留存证据。

4. **出具整改通知**
   - 编写《不符合项报告》，内容包括：问题描述、风险等级（严重/重要/一般）、整改建议、整改时限。
   - 风险等级定义：
     - **严重**：可能导致数据泄露、法律处罚（如：无权限管控的批量导出）。**整改完成前系统禁止上线**。
     - **重要**：存在安全隐患但未直接触发（如：日志不完整）。整改时限30天。
     - **一般**：流程不规范但无实际危害（如：制度未及时更新）。整改时限90天。
   - 通知发送给审计对象负责人，并抄送合规/安全部门。
   - **升级路径**：若审计对象对整改通知有异议，或 [待确认升级处理流程]

5. **整改复核**
   - 审计对象完成整改后，提交整改证据（如：配置变更截图、新增审批流程截图）。
   - 复核整改是否到位：对照《不符合项报告》逐条验证。
   - 若复核不通过，退回整改并重新设定整改时限；若复核通过，进入归档。

6. **结案归档**
   - 整理全套审计文档：审计计划、范围标准确认记录、证据包、穿行测试记录、不符合项报告、整改证据、复核记录。
   - 归档至 [待确认归档系统/路径]。
   - 发送结案通知给审计发起方和审计对象负责人。

## Decision routes
| 用户意图/条件 | 执行动作 | 备注 |
|------------|---------|------|
| 用户说“开始审计”或“收到审计任务” | 执行步骤1：接收审计任务与初步沟通 | 先收集基础资料 |
| 审计对象对整改通知有异议 | 执行升级路径：[待确认] | 需补充升级处理流程 |
| 整改复核不通过 | 退回整改，重新设定整改时限 | 记录退回原因 |
| 整改复核通过 | 执行步骤6：结案归档 | 发送结案通知 |

## Inputs
- audit_source: string, 必填, 审计发起方（如：合规部、安全部、业务方）
- audit_target: string, 必填, 审计对象（系统名/部门名/流程名）
- audit_background: string, 必填, 审计背景（合规检查/事件响应/例行抽查）
- urgency: enum, 默认"常规", 紧急程度（紧急/常规）
- deadline: string, 必填, 完成时限（如：2024-01-15）
- audit_scope: string, [待确认], 审计范围详细描述
- compliance_standard: string, [待确认], 合规标准列表

## Outputs
- status: enum — 执行结果（success / pending / rejected / escalated）
- message: string — 给用户或下游系统的摘要说明
- audit_notes: string — 关键操作记录（可选）
