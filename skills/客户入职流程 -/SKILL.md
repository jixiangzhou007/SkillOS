# 客户入职流程 - 新客户签约后建workspace

## 核心问题
销售签约后，自动完成 workspace 创建、权限配置、通知培训、回访归档，确保客户顺利入职。

## When to use
- keywords: 客户入职, 建 workspace, 新客户签约, 入职流程, 客户 onboarding
- context: 销售团队完成签约后，需要为新客户创建 workspace 并完成后续入职步骤
- excludes: 客户续约、客户升级套餐（非首次入职）、客户投诉处理

## Instructions
Follow these steps in order. Ask the user if anything is marked [待确认].

1. **销售触发入职流程**
   - 销售在系统内点击“触发入职”按钮，或通过邮件/消息通知入职团队。
   - 系统自动创建入职任务，状态设为“待信息收集”。

2. **信息收集与验证**
   - 收集以下必填信息：客户名称、公司名、联系人、合同人数、套餐类型。
   - 验证信息完整性：若缺少任意必填项，标记为“信息不完整”并通知销售补充。
   - 若信息完整，生成 `客户入职信息单`（结构化数据）。

3. **创建 Workspace（6 步之一）**
   - 根据套餐类型和人数，在后台系统创建新的 Workspace 实例。
   - 分配初始资源（存储、计算、用户席位）。
   - 配置基础网络、域名、SSL 证书。
   - 若步骤 2 存在“信息不完整”标记，阻塞此步骤，直到问题解决。

4. **发送入职邮件（6 步之二）**
   - 向客户指定的管理员邮箱发送入职欢迎邮件。
   - 邮件内容包含：Workspace 访问链接、初始管理员账号、快速入门指南。
   - 抄送销售和客户成功经理。

5. **安排培训（6 步之三）**
   - 根据套餐类型，自动推荐培训课程（标准版：录播教程；企业版：1 对 1 在线培训）。
   - 创建培训任务，分配给客户成功团队。
   - 若客户有特殊培训需求，标记为 [待确认] 并通知客户成功经理。

6. **客户回访（6 步之四）**
   - 入职后第 3 天，自动触发首次回访任务。
   - 回访内容：确认 Workspace 使用情况、收集反馈、解答疑问。
   - 若客户反馈问题，创建工单并分配给技术支持。

7. **归档（6 步之五）**
   - 入职后第 7 天，确认所有步骤完成。
   - 将 `客户入职信息单`、沟通记录、培训记录归档到客户档案。
   - 关闭入职任务，状态设为“已完成”。

8. **异常处理**
   - 若任何步骤失败（如创建 Workspace 失败、邮件发送失败），系统自动重试 2 次。
   - 重试仍失败，标记为“异常”，通知入职团队人工介入。
   - 若客户在入职过程中提出变更需求（如修改套餐、增加人数），暂停流程，等待销售确认后继续。

## Decision routes
| 用户意图/条件 | 执行动作 | 备注 |
|------------|---------|------|
| 销售触发入职 | 开始步骤 1，创建入职任务 | 流程入口 |
| 信息不完整 | 通知销售补充，阻塞后续步骤 | 避免错误创建 |
| 信息完整 | 继续步骤 3 创建 Workspace | 正常流程 |
| 创建 Workspace 失败 | 重试 2 次，失败则通知人工 | 异常处理 |
| 客户提出变更需求 | 暂停流程，等待销售确认 | 边界情况 |
| 入职后第 3 天 | 触发回访任务 | 自动定时任务 |
| 入职后第 7 天 | 归档并关闭任务 | 流程结束 |

## Inputs
- customer_name: string, 必填, 客户名称
- company_name: string, 必填, 公司名
- contact_person: string, 必填, 联系人姓名
- contact_email: string, 必填, 联系人邮箱
- headcount: integer, 必填, 合同人数
- plan_type: enum("标准版", "企业版"), 必填, 套餐类型
- special_requirements: string, 可选, 特殊备注或需求
