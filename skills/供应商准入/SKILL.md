# 供应商准入

## 核心问题
标准化供应商从候选到入库的准入流程，确保资质合规、法律风险可控、评审透明。

## When to use
- keywords: 供应商准入, 供应商入库, 供应商审核, 供应商资质, 供应商评审
- context: 采购提交候选供应商信息时自动触发，用于标准化供应商从候选到入库的准入流程
- excludes: 供应商日常管理、供应商绩效评估、供应商淘汰退出流程

## Instructions
Follow these steps in order. Ask the user if anything is marked [待确认].

1. 资质预审：收集候选供应商的营业执照和行业资质证书。
   - 检查资质是否在有效期内，若过期则标记为“资质过期”，流程终止并发送拒信。
   - [待确认] 对于即将到期的资质（有效期不足30天），处理策略待确认（选项A：视为过期直接终止；选项B：允许进入后续流程，但入库前必须完成续期）。
   - [待确认] 若供应商分批提交资质文件（如先提交行业资质，后提交营业执照），处理策略待确认（选项A：所有文件到齐后启动预审；选项B：先校验已到资质，致命问题可提前终止）。
2. 法律风险核查：通过天眼查等工具查询供应商是否存在法律诉讼、经营异常等风险。
   - 若存在失信被执行或重大诉讼，直接终止流程并发送拒信。
   - [待确认] 风险等级判定标准（高/中/低）及对应处理策略。
3. 信息填报：法律风险核查通过后，发送供应商基本信息表，要求填写企业规模、主要客户、银行账户等信息。
   - [待确认] 基本信息表模板或必填字段清单。
4. 内部评审：收集完整信息后，发起内部评审流程。
   - 评审组由采购、法务、质量三方组成。任何一方否决，直接判定不通过，发送拒信。
   - [待确认] 评审流转方式（邮件审批/系统流转/会议评审）。
   - [待确认] 评审时限（每个评审节点需在X个工作日内完成）。
   - [待确认] 评审维度细化（除资质合规性、法律风险外，是否包含交付能力、价格竞争力等）。
5. 入库或拒信：评审通过则加入供应商库，不通过则发送拒信。
   - [待确认] 供应商库的信息存储结构（如供应商ID、评级、联系人等）。
   - [待确认] 拒信发送方式（邮件/系统通知）及模板。

## Decision routes
| 用户意图/条件 | 执行动作 | 备注 |
|------------|---------|------|
| 资质过期 | 终止流程，发送拒信 | 要求供应商提供最新资质后重新申请 |
| 法律风险核查发现失信/重大诉讼 | 终止流程，发送拒信 | 直接淘汰，不入库 |
| 法律风险核查通过 | 发送基本信息表，进入信息填报环节 | 等待供应商填写并提交 |
| 内部评审任何一方否决 | 判定不通过，发送拒信 | 采购/法务/质量三方任一否决即不通过 |
| 内部评审全部通过 | 加入供应商库，流程结束 | 记录供应商信息及评级 |

## Inputs
- supplier_name: string, 必填, 候选供应商名称
- business_license: file, 必填, 营业执照扫描件或照片
- qualification_certificates: file[], 必填, 行业资质证书列表
- tianyancha_report: file, 必填, 天眼查等工具出具的法律风险报告
- basic_info_form: object, [待确认], 供应商基本信息表（企业规模、主要客户、银行账户等）
- review_result: enum, [待确认], 内部评审结果（通过/否决）
- final_status: enum, [待确认], 最终状态（入库成功/拒信已发）

## Outputs
- status: enum — 执行结果（success / pending / rejected / escalated）
- message: string — 给用户或下游系统的摘要说明
- audit_notes: string — 关键操作记录（可选）
