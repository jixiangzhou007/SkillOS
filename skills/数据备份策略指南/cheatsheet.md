# 速查表: 数据备份策略指南

```markdown
# 数据备份策略速查表

## 核心流程（六步法）
1. **范围界定** → 2. **压缩** → 3. **加密** → 4. **传输** → 5. **验证** → 6. **清理**

> **关键规则：先压缩，后加密**（压缩后加密效率高，加密后压缩几乎无效）

## 决策流程（if-then 分支）

| 场景 | 操作 |
|------|------|
| 备份范围不确定 | 优先排除日志、缓存等临时数据，只保留数据库转储、配置、用户上传 |
| 需要压缩 | 使用 `zstd`，可指定压缩级别（如 `--level 3`） |
| 需要加密 | 使用 `AES-256-GCM`，密钥从外部服务（如 Vault）动态获取 |
| 需要传输 | 上传至 S3 等对象存储，**必须附带 MD5 校验和** |
| 传输完成 | 立即验证完整性，失败则触发告警 |
| 需要清理 | 按分层保留策略删除过期备份（见下方数字阈值） |
| 备份影响性能 | 安排在业务低峰时段执行 |

## 常见陷阱（避免做的事）
- ❌ **未加密传输**：数据在网络上明文传输，易被截获或篡改
- ❌ **密钥硬编码**：将加密密钥写在脚本或配置文件中
- ❌ **高峰时段备份**：导致数据库或系统性能下降，影响正常服务
- ❌ **跳过验证**：传输后不校验完整性，损坏数据无法及时发现
- ❌ **保留策略混乱**：不按时间分层，导致存储成本过高或恢复点不足

## 关键数字/阈值

| 备份类型 | 保留周期 |
|----------|----------|
| 每日备份 | 7 天 |
| 每周备份 | 4 周 |
| 每月备份 | 12 个月 |

- 压缩工具：`zstd`
- 加密算法：`AES-256-GCM`
- 校验方式：`MD5` 校验和
- 密钥管理：`Vault`（外部化）

## 一句话总结
> **备份必须覆盖“范围→压缩→加密→传输→验证→清理”六个环节，并避开未加密传输、密钥硬编码、高峰执行等陷阱，才能兼顾安全与可操作。**
```
