# 技能名称：套飞书报销审批的处理流程
## 触发场景
- 员工在飞书提交「费用报销」模板
- 附件必须包含：发票 PDF + 付款截图

## 输入/前置条件
- 飞书审批单（含申请人部门、费用类型、金额）
- 发票 PDF（需为真实发票，抬头为公司全称）
- 付款截图（显示实际付款金额与收款方）
- [待补充：发票号（用于重复报销核验）]
- [待补充：紧急预付款标识（用于触发预付款流程）]

## 执行步骤（按顺序）
### 第一步：基础信息核对
- **核对内容**：申请人部门、费用类型（差旅/办公/招待）、金额、发票抬头
- **发票抬头要求**：必须为公司全称（如“XX创新科技有限公司”）
- **异常处理**：
  - 发票抬头不符 → 退回修改，要求重开发票（高风险）
  - 付款截图收款方与发票收款方不一致 → [待补充：需判断是否同一实体，如不一致则标记异常]
  - 金额与发票金额不一致 → [待补充：退回或要求说明]

### 第二步：金额与权限校验
- **规则**：
  - 0 - 500元（不含500元）：部门负责人审批
  - 500元 - 5,000元（含500元，不含5,000元）：CFO 审批
  - 5,000元及以上（含5,000元）：CFO 审批后，CEO 加签
- **特殊规则**：
  - 差旅费必须附带行程单
  - 招待费必须写清楚客户名称
- **异常处理**：
  - [待补充：如果金额超限但审批人不在线或拒绝，流程如何处理]

### 第三步：财务复核与闭环
- **内容**：财务核对发票真伪、付款截图一致性、验证发票号是否重复（系统自动拦截并通知财务）
- **紧急预付款处理**：若标记为“紧急预付款”，则跳过发票核验，直接进入预付款流程，事后补发票
- **凭证生成**：审批通过后，生成会计凭证并归档至电子台账
- **输出**：飞书审批通过/拒绝/退回记录 + 电子台账凭证号

## 输出/产出
- 飞书审批通过/拒绝/退回记录
- 电子台账凭证号（用于财务对账）

## 边界情况
- [待补充：发票丢失、付款截图不清晰、跨部门报销、金额恰好等于500元或5000元时归属哪个区间]
- 重复报销：同一发票号系统自动拦截并通知财务
- 紧急采购：可先走「预付款」流程，事后补发票（需在审批单中标记“紧急预付款”）
