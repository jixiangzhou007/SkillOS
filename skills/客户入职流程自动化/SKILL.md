# 客户入职流程自动化

## 核心问题
新客户签约后，如何自动完成建workspace、发欢迎邮件、预约培训、7天回访和归档，确保客户顺利入职并降低流失风险。

## When to use
- keywords: 客户入职, 签约, 新客户, onboarding, workspace, 欢迎邮件, 培训, 回访, 归档
- context: 当销售在CRM中将客户状态更新为“已签约”时，自动触发客户入职流程。用户可能说“帮我沉淀客户入职流程”、“新客户签约后怎么做”等。
- excludes: 客户尚未签约、客户是已有客户重新激活、客户是内部测试账号

## Instructions
Follow these steps in order. Ask the user if anything is marked [待确认].

1. **触发流程**：销售在CRM中将客户状态更新为“已签约”，系统自动获取以下输入：客户名、公司名、人数、套餐、合同PDF。
2. **步骤1：建Workspace并选套餐**
   - 在系统中为客户创建专属workspace。
   - 套餐选择：如果CRM中销售已填写套餐字段，直接应用；如果未填写，流程暂停并触发通知（通知对象[待确认]：销售/CSM）。
   - 边界条件：如果客户有特殊定制需求，套餐标记为“定制待确认”；如果销售口头承诺但未记录，触发提醒销售补充备注；如果CRM中套餐字段为空或合同PDF未上传，流程暂停并触发通知。
3. **步骤2：发送欢迎邮件**
   - 触发条件：workspace创建完成且套餐已选定。
   - 邮件内容：包含客户名、公司名、套餐名等个性化信息，以及workspace链接和临时密码。
   - 抄送：销售负责人、客户成功经理。
   - 边界条件：如果客户3天内未登录workspace，自动重发欢迎邮件（内容不变，仅更新链接有效期）。
4. **步骤3：预约培训**
   - 触发条件：欢迎邮件发送后。
   - 动作：向客户发送培训预约链接（或由客户成功经理手动约时间）。
   - 边界条件：如果客户7天内未响应，[待确认]是否自动触发第二次提醒。
5. **步骤4：7天回访**
   - 触发条件：培训完成或客户首次登录后第7天。
   - 动作：客户成功经理进行回访，记录反馈。
   - 回访打分：1-5分，如果低于3分，升级客户成功经理处理。
   - 输出：回访记录写入CRM。
6. **步骤5：归档**
   - 触发条件：回访完成且客户已正常使用产品超过14天（或双方确认无后续问题）。
   - 动作：将客户入职状态标记为“已完成”，相关文档归档。
   - 边界条件：如客户在14天内提出投诉或问题未解决，归档延迟直至问题关闭。

## Inputs
- customer_id: string, 必填, CRM中的客户唯一标识
- customer_name: string, 必填, 客户名称
- company_name: string, 必填, 公司名称
- employee_count: integer, 必填, 客户公司人数
- plan: string, 可选, 套餐名称（如果CRM中已填写则使用，否则流程暂停）
- contract_pdf: file, 必填, 合同PDF文件
- sales_owner: string, 必填, 销售负责人
- csm_owner: string, 可选, 客户成功经理（默认由系统分配）
