# 工单分类处理

## 核心问题
这个流程解决如何对客户提交的技术支持工单进行自动分类、分派和SLA跟踪，确保不同类型的请求得到正确的处理路径。

## When to use
- keywords: 工单, Zendesk, 技术支持, 故障, 咨询, 功能请求, SLA, 升级, 知识库, FAQ
- context: 当客户通过邮件或 Zendesk 工单系统提交问题，需要自动分类和处理时使用
- excludes: 非技术类客服工单（如账单、账户管理）、已由人工处理的工单、内部测试工单

## Instructions
Follow these steps in order. Ask the user if anything is marked [待确认].

1. 识别工单类型：
   - 读取工单标题和描述内容，判断属于以下三类之一：
     - **故障**：系统报错、服务不可用、功能异常等
     - **咨询**：如何使用、配置说明、最佳实践等
     - **功能请求**：希望新增功能、改进现有功能等
   - 如果无法明确分类，标记为「待人工确认」并转人工客服

2. **信息完整性检查（新增）**：
   - 检查工单中是否包含关键排障信息：日志文件、错误截图、产品版本号
   - 如果信息缺失：
     - 对于P1/P2故障：**立即发送模板邮件索要缺失信息，同时并行执行后续处理步骤** [待确认：模板邮件具体内容]
     - 对于P3故障/咨询/功能请求：先发送模板邮件索要信息，待客户回复后再继续处理
   - 如果信息完整，直接进入对应类型的处理流程

3. 故障类处理：
   - 立即在 Confluence 知识库中搜索相关故障解决方案
   - 如果 30 分钟内找到匹配方案 → 回复客户并关闭工单
   - 如果 30 分钟内未找到方案 → 升级至二线工程师团队，并通知客户

4. 咨询类处理：
   - 在 FAQ 库中匹配客户问题
   - 如果匹配度 ≥ 80% → 自动回复 FAQ 答案并关闭工单
   - 如果匹配度 < 80% → 转接人工客服处理

5. 功能请求类处理：
   - 记录客户需求详情（功能描述、使用场景、期望优先级）
   - 转交产品团队进行需求评估
   - 不计入故障 SLA 统计

6. SLA 响应时效跟踪：
   - P1（紧急故障）：4 小时内首次响应
   - P2（普通故障）：当天（24 小时内）首次响应
   - P3（咨询/其他）：3 个工作日内首次响应
   - 超时未响应 → 自动升级通知 [待确认：升级通知的具体接收人]

## Decision routes
| 用户意图/条件 | 执行动作 | 备注 |
|------------|---------|---
| 工单信息缺失且为P1/P2 | 并行：发模板邮件 + 查知识库 | 不因等待信息而延误P1/P2响应 |
| 工单信息缺失且为P3/咨询/功能请求 | 先发模板邮件，等待回复 | 低优先级可等待信息完备后再处理 |
