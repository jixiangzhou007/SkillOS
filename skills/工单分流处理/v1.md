# 工单分流处理

## 核心问题
这个流程解决客户工单的自动分类、分流、升级和关单确认问题，确保故障、咨询、功能请求三类工单按不同路径高效处理。

## When to use
- keywords: 工单, 客服, 故障, 咨询, 功能请求, SLA, Zendesk, 邮件, 支持, 问题, 客户
- context: 处理客户通过邮件或Zendesk工单系统提交的问题时使用
- excludes: 非客户工单处理场景（如内部任务、项目管理、开发任务），或客户已明确要求人工处理

## Instructions
Follow these steps in order. Ask the user if anything is marked [待确认].

1. 识别工单类型：根据客户描述判断属于故障、咨询、功能请求三类之一。
   - 若无法明确分类，标记为「待确认」并转人工客服处理。
2. 若工单类型为故障：
   a. 首先在 Confluence 知识库中搜索已知解决方案。
   b. 如果在 30 分钟内找到匹配方案，直接回复客户并执行下一步。
   c. 如果 30 分钟内未找到方案，将工单升级至二线工程师处理。
3. 若工单类型为咨询：
   a. 在 FAQ 库中匹配客户问题。
   b. 如果匹配度 ≥ 80%，直接回复 FAQ 答案。
   c. 如果匹配度 < 80%，将工单转接至人工客服处理。
4. 若工单类型为功能请求：
   a. 记录客户需求详情（功能描述、期望效果、业务场景）。
   b. 将需求转交至产品团队。
   c. 此工单不计入故障 SLA 统计。
5. 根据工单优先级执行 SLA 响应：
   - P1（紧急故障）：4 小时内首次响应。
   - P2（一般故障）：当天内首次响应。
   - P3（低优先级）：3 个工作日内首次响应。
6. 信息补全：如果客户提交的信息不完整（缺少日志、截图、产品版本号等），使用模板邮件向客户索取缺失信息，等待客户回复后再继续处理。
7. 关单确认：在关闭工单前，必须获得客户明确确认问题已解决。如果客户未确认，保持工单状态为 open，并定期跟进（[待确认]跟进频率）。
