# 技能名称：订单提交后的字段提取与校验流程

## 触发场景
- 当用户提交订单时触发

## 输入/前置条件
- 订单ID（必填）
- 客户ID（必填）
- [待补充] 订单数据结构（如：商品ID、数量、用户ID）
- [待补充] 数据源（如：商品数据库、用户数据库、退款政策数据库）

## 执行步骤
- 第1步：验证订单
  - [待补充] 检查输入字段是否存在（如：商品ID字段是否为空或缺失）
  - [待补充] 校验字段值的合法性（如：数量不能为负数或零）
  - [待补充] 校验数据源一致性（如：商品ID在数据库中是否存在）
- 第2步：检查退款政策
  - [待补充] 根据订单中的商品ID或商品类别，查询对应的退款政策
  - [待补充] 判断该订单是否满足退款条件（如：购买时间、商品状态）
- 第3步：计算金额
  - [待补充] 根据退款政策，计算应退金额（如：全额退款、按比例退款、扣除手续费）
- 第4步：退款
  - [待补充] 执行退款操作（如：调用支付网关API）
  - [待补充] 更新订单状态为“已退款”
  - [待补充] 无论成功或失败，均发送邮件通知用户

## 输出/产出（待探索）
- [待补充] 提取成功：返回结构化数据（如：商品名称、单价、库存）
- [待补充] 提取失败：返回错误码与错误描述
- [待补充] 校验异常：返回风险等级与处理建议

## 边界情况（待探索）
- [待补充] 字段缺失：如何处理订单中缺少必要字段（如商品ID为空）
- [待补充] 数据匹配失败：商品ID在数据库中不存在
- [待补充] 字段值异常：数量为负数或超出库存限制
- [待补充] 退款政策不明确：商品对应的退款政策不存在或已过期
- [待补充] 金额计算异常：计算结果为负数或超出订单总金额
