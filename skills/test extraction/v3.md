# 技能名称：test extraction flow 设计诊断

## 触发场景
- 用户提交订单时触发
- 需要从订单数据中提取特定测试用例字段

## 输入/前置条件
- [待补充] 订单数据结构定义（字段名、类型、可选性）
- [待补充] 提取规则映射表（源字段 → 目标字段）
- **[新增] 订单状态枚举值列表（如：pending, confirmed, cancelled, completed）**

## 执行步骤
1. 接收订单数据（JSON/对象格式）
2. 根据映射表提取目标字段
3. 对提取结果进行结构校验
4. 输出结构化测试数据

## 输出/产出
- [待补充] 输出格式（JSON Schema 或 CSV 表头）
- [待补充] 输出字段列表

## 边界情况
- [待补充] 字段缺失时的默认值/空值处理
- [待补充] 字段值格式异常（如日期格式不统一）的处理
- [待补充] 同一字段在不同订单中映射不一致的检测机制
- [待补充] 字段值为 null 或空字符串时的处理策略
  - 当前需确认：是否统一跳过、标准化为空字符串、或报错终止
- **[新增] 订单状态为“已取消”或“异常”时的处理策略**
  - 当前需确认：是否应作为前置校验条件，在步骤1前拦截，或作为特殊分支继续执行
