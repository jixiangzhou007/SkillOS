# 技能名称：退款处理
## 核心问题
规范淘宝小店退款申请的处理流程，确保订单核对、政策执行、边界处理及后续操作标准化。

## S_body
1. **核对订单信息**：在旺旺或订单页获取买家申请的退款单（仅退款/退货退款），核对订单号、实付金额、付款渠道（支付宝/微信），并在 ERP 中查询发货状态和物流签收时间。
2. **按状态执行退款政策**：
   - if 未发货 → 执行秒退（直接同意退款）。
   - if 已发货且未签收 → 先联系快递公司发起拦截，拦截成功后退款；若拦截失败，转主管审批。
   - if 已签收 → 走七天无理由流程，检查商品完好性（需买家提供凭证）；若商品完好，同意退款；若商品损坏，转主管审批。
3. **特殊规则判断**：
   - if 退款金额 > 500元 → 转主管审批，并在飞书群留截图记录。
   - if 申请时间超过签收后7天 → 转主管审批。
   - if 买家威胁差评 → 升级处理（可转主管），但不代表必须全额退款。
   - if 商品为虚拟商品（下载链接类）且已发货 → 不支持退款，直接拒绝并说明原因。
   - if 申请类型为换货 → 不属于退款流程，转另一个工单类型处理。
4. **标记与后续操作**：
   - 在 ERP 中标记退款原因分类（可选值：质量问题/不喜欢/发错货）。
   - 退款完成后24小时内，通过旺旺发送一条模板消息确认退款结果。
   - 同步更新 CRM 备注，记录退款原因及处理结果。
