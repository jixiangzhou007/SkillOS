# Test
## S_body
1. 核对订单信息：确认订单号、用户身份、购买商品及金额是否与退款申请一致。
2. 按订单状态分流：
   - 若订单为“未发货”，直接进入“待退款”队列，标记为“可全额退款”，**适用秒退规则（无需人工审核，系统自动处理）**。
   - 若订单为“已发货”，检查物流状态：
     - 若物流显示“未签收”，联系用户确认是否拦截退回，拦截成功则进入“待退款”队列。
     - 若物流显示“已签收”，转入“人工审核”流程，需核实退货商品是否寄回。
3. 边界情况处理：
   - **时限判断**：若订单已签收且当前申请时间超过签收后7天，提示“超出退款时限（签收后7天）”，拒绝申请并记录原因。
   - **金额判断**：若退款金额>500元，转入“主管审批”流程，需主管二次确认后方可进入后续处理。
   - **零金额订单**：若订单金额为0（如赠品或积分兑换），提示“零金额订单不支持退款”，拒绝申请。
   - **虚拟商品**：若商品为虚拟商品（如电子卡券、数字内容、**下载链接类**）且订单状态为“已发货”，提示“虚拟商品已发货不支持退款”，拒绝申请；**若为下载链接类且未发货，可支持退款**。
   - **换货场景**：若用户申请类型为“换货”，转入“**换货工单**”流程（工单类型为“售后换货”），不按退款逻辑处理。
   - **异常行为**：若用户多次发起退款（同一订单超过3次），标记为“异常订单”，转人工审核。
   - **买家威胁差评**：若用户以“给差评”为条件要求退款，**立即升级至主管处理**，主管根据实际情况决定是否退款或给予补偿，并在日志中记录“威胁差评”标签。
4. 记录要求：将每一步操作结果（包括订单号、处理时间、处理人、最终状态）写入系统日志，确保可追溯。
