# 客户入职流程

## 核心问题
新客户签约后，如何自动完成工作空间创建、欢迎邮件、培训安排、回访确认和归档，确保入职流程标准化、可追踪。

## When to use
- keywords: 客户入职, 签约, 新客户, onboarding, 入职流程, 客户上线
- context: 销售CRM中客户状态被标记为“已签约”后自动触发，或用户手动要求执行客户入职流程时使用
- excludes: 客户尚未签约、合同未签署、客户信息不完整（触发补全流程前）、客户处于退款/取消流程中

## Instructions
Follow these steps in order. Ask the user if anything is marked [待确认].

1. 接收签约完成信号：
   - 触发条件：销售CRM中将客户状态标记为“已签约”。
   - 必填字段校验：客户名、公司名、人数、合同附件。若任一缺失，触发“信息补全流程”（见边界情况）。
2. 创建工作空间（Workspace）：
   - 根据客户信息自动生成workspace名称（格式：客户名_签约日期）。
   - 若缺少「客户行业」或「预计使用人数」，按配置策略处理（[待确认：具体策略，例如：人数缺失时默认5人，或暂停流程通知销售]）。
   - 分配默认管理员角色（[待确认：默认管理员邮箱]）。
   - 记录workspace ID回写至客户档案。
3. 发送欢迎邮件：
   - 邮件模板：[待确认：欢迎邮件模板链接或内容]
   - 抄送客户成功经理（CSM）和销售代表。
   - 若workspace创建失败，触发告警并暂停邮件发送。
4. 安排培训：
   - 根据客户行业/人数，推荐培训套餐（[待确认：培训套餐映射表]）。
   - 生成培训邀请链接，随邮件一并发送。
   - 若客户未在48小时内点击培训链接，触发回访提醒。
5. 回访确认：
   - 培训完成7天后，自动发送满意度问卷。
   - 若问卷评分低于3分，自动创建回访工单（工单类型：[待确认]）。
6. 归档：
   - 所有步骤完成后，将客户状态标记为“已入职”。
   - 生成入职报告（[待确认：报告模板]），推送至归档存储（[待确认：存储路径]）。

## Decision routes
| 用户意图/条件 | 执行动作 | 备注 |
|------------|---------|------|
| 客户信息缺失（客户名/公司名/人数/合同附件） | 向销售CRM负责人发送补全通知，设置24小时超时 | 超时策略：[待确认：例如超时后使用默认值，或升级通知销售主管] |
| workspace创建失败 | 触发告警，记录错误日志，暂停后续流程 | 不发送邮件、不安排培训 |
| 邮件发送失败 | 重试3次，仍失败则创建工单通知CSM | 重试间隔：[待确认] |
| 客户拒绝培训 | 在回访环节标注“客户拒绝”，归档时计入备注 | 跳过培训步骤 |
| 客户在入职期间提出修改合同 | 暂停流程，转销售跟进 | 转交规则：[待确认] |

## Inputs
- customer_name: string, 必填, 客户名称
- company_name: string, 必填, 客户公司全称
- employee_count: integer, 必填, 客户公司人数
- contract_attachment: file, 必填, 已签署的合同文件
- industry: string, 可选, 客户所属行业（缺失时按策略处理）
- estimated_users: integer, 可选, 预计使用人数（缺失时按策略处理）
- default_admin_email: string, 可选, 默认管理员邮箱（[待确认]）
