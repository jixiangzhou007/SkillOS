# 供应商评估

## 核心问题
标准化供应商评估流程，确保从资质审查到合同归档的每个环节可追溯、可量化，避免主观判断和流程遗漏。

## When to use
- keywords: 供应商评估, 供应商评分, 供应商选择, 供应商准入, 供应商评审, 供应商考察, 供应商比选
- context: 用户需要标准化供应商评估流程，从资质审查到合同归档的完整闭环
- excludes: 用户仅询问供应商信息查询、供应商日常管理、供应商绩效评估（非首次准入评估）

## Instructions
Follow these steps in order. Ask the user if anything is marked [待确认].

1. **资质审查**
   - **输入**：候选供应商名单、资质文件（营业执照、认证证书、过往合同案例等）、报价单
   - **行动**：
     - 核查工商信息（注册时间、注册资本、实缴资本、经营范围、股东背景）
     - 核查行业口碑（同行评价、过往项目案例、是否有负面新闻/诉讼）
     - 核查资质证书（如ISO认证、行业许可证等）
   - **决策规则**：
     - 若存在失信被执行人、重大诉讼等重大负面信息 → 直接判定为高风险，终止流程，通知采购并说明原因
     - 若成立时间<1年或注册资本<100万 → 需进一步核查股东背景和实缴资本，不能仅凭口碑或注册时间直接判定
     - 若资料不完整 → 设置3个工作日SLA等待供应商补充，超时自动判定为不通过
   - **输出**：资质审查通过/不通过/需补充材料

2. **报价与商务条款对比**
   - **输入**：各供应商报价单、商务条款（付款方式、交货期、质保期、违约责任等）
   - **行动**：
     - 制作报价对比表，列出各供应商的价格、折扣、付款条件
     - 对比商务条款差异，标记异常条款（如付款比例过高、质保期过短）
   - **输出**：报价对比表、商务条款差异分析

3. **技术评审**
   - **输入**：供应商技术方案、产品规格书、案例演示
   - **行动**：
     - 组织技术评审会议，邀请相关技术人员参与
     - 评估方案可行性、技术成熟度、与现有系统兼容性
     - 记录技术评审意见和评分
   - **输出**：技术评审报告（含评分和意见）

4. **综合评分**
   - **评分维度与权重**：
     - 资质：20分
     - 价格：30分
     - 技术：30分
     - 服务：20分
   - **决策规则**：
     - 总分低于60分的供应商直接淘汰，不留底
   - **输出**：综合评分表（各维度得分+总分+排名）

5. **推荐中标**
   - **行动**：根据综合评分排名，推荐得分最高的供应商为中标候选
   - **输出**：中标推荐报告（含评分依据和推荐理由）

6. **合同签署与归档**
   - **输入**：中标供应商合同文本
   - **行动**：
     - 确认合同条款与报价/商务条款一致
     - 推动双方签署
     - 归档合同原件及评估全流程文档
   - **输出**：已签署合同、完整评估档案

## Decision routes
| 用户意图/条件 | 执行动作 | 备注 |
|------------|---------|------|
| 采购提交候选供应商名单 | 启动资质审查（步骤1） | 需确认触发方式（邮件/工单/表单）[待确认] |
| 资质审查发现重大负面信息 | 终止流程，通知采购并说明原因 | 高风险供应商直接淘汰 |
| 资质审查资料不完整 | 设置3个工作日SLA等待补充，超时自动不通过 | 需确认SLA天数是否可配置[待确认] |
| 综合评分低于60分 | 直接淘汰，不留底 | 淘汰供应商不进入后续环节 |
| 综合评分完成，有供应商达标 | 推荐得分最高者中标，进入合同签署 | 若多个供应商均达标，按排名推荐 |
| 合同签署完成 | 归档全流程文档 | 确保评估过程可追溯 |

## Inputs
- 供应商名单: array<string>, 必填，候选供应商名称列表
- 资质文件: array<file>, 必填，营业执照、认证证书、过往合同案例等
- 报价单: array<file>, 必填，各供应商报价文件
- 资质审查阈值: object, {成立时间: "≥1年", 注册资本: "≥100万"}，工商合格判定标准
- 高风险标记条件: array<string>, ["失信被执行人", "当前重大诉讼"]，直接终止流程的条件
- 需补充材料SLA: number, 3，单位工作日，超时自动不通过
- 评分维度: object, {资质: 20, 价格: 30, 技术: 30, 服务: 20}，各维度满分
- 淘汰分数线: number, 60，低于此分数直接淘汰
