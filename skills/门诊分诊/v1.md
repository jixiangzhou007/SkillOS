# 门诊分诊

## 核心问题
快速、准确地将门诊患者按病情严重程度分级，并引导至正确的科室或抢救区域，避免延误治疗。

## When to use
- keywords: 分诊，门诊分诊，病情分级，安排科室，挂号，测体温，测血压，主诉，病史，抢救室，候诊
- context: 用户正在处理门诊患者的分诊工作，需要根据患者情况快速决策。
- excludes: 患者已明确诊断为非急症且无需分诊；用户仅咨询挂号流程而非病情评估。

## Instructions
Follow these steps in order. Ask the user if anything is marked [待确认].

1.  患者挂号后，首先进行基础体征测量：测量体温和血压。
2.  询问患者的主诉（主要症状）与病史（既往病史、过敏史等）。
3.  根据体征与病史进行病情分级：
    - **一级（濒危/生命不稳）**：患者生命体征不稳定，如心跳骤停、严重呼吸困难、休克等。
        - **动作**：立即送入抢救室。
    - **二级（危重/危险）**：患者有潜在生命危险，如急性心梗、严重创伤、大出血等，但生命体征暂时稳定。
        - **动作**：安排患者在 **15分钟内** 见到医生。
    - **三级（急症）**：患者有急性症状，但无生命危险，如急性腹痛、骨折、高热等。
        - **动作**：安排候诊，按顺序就诊。
    - **四级（非急症）**：患者症状轻微，如普通感冒、慢性病复诊、皮肤小问题等。
        - **动作**：安排候诊，可引导至普通门诊或专科门诊。
4.  根据分级结果，将患者安排至对应科室或区域就诊。

## Decision routes
| 用户意图/条件 | 执行动作 | 备注 |
|------------|---------|------|
| 患者生命体征不稳定（一级） | 立即送入抢救室 | 无需等待 |
| 患者有潜在生命危险（二级） | 安排15分钟内见医生 | 优先处理 |
| 患者有急性症状但无生命危险（三级） | 安排候诊 | 按顺序就诊 |
| 患者症状轻微（四级） | 安排候诊或引导至普通门诊 | 可适当延后 |
| 患者体温或血压异常 | [待确认] 具体处理标准（如：体温>39℃或血压>180/110mmHg是否升级为二级） | 需补充异常值阈值 |
| 患者主诉模糊 | [待确认] 追问方式（如：询问“最不舒服的部位是哪里？”“症状持续多久了？”）或根据已知病史分流 | 需补充追问话术 |

## Inputs
- 体温: float，默认值 36.5，单位 ℃，说明：患者当前体温。
- 收缩压: int，默认值 120，单位 mmHg，说明：患者收缩压。
- 舒张压: int，默认值 80，单位 mmHg，说明：患者舒张压。
- 主诉: string，默认值 ""，说明：患者自述的主要症状。
- 病史: string，默认值 ""，说明：患者的既往病史和过敏史。
- 分级结果: string，默认值 "四级"，可选值 ["一级", "二级", "三级", "四级"]，说明：根据体征和病史判断的病情等级。
- 安排科室: string，默认值 "候诊区"，说明：根据分级结果安排的就诊区域或科室。

## Outputs
- status: enum — 执行结果（success / pending / rejected / escalated）
- message: string — 给用户或下游系统的摘要说明
- audit_notes: string — 关键操作记录（可选）
