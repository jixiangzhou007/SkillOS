# 门诊分诊

## 核心问题
快速、标准地对门诊患者进行病情评估和科室分配，避免误诊漏诊和资源浪费。

## When to use
- keywords: 门诊分诊, 分诊, 患者分流, 急诊分级, 预检分诊
- context: 当用户需要根据患者症状和体征快速评估病情严重程度并分配就诊科室时使用。适用于医院门诊、急诊预检、基层医疗分诊场景。
- excludes: 不适用于住院患者的分级护理、手术分级、药物分级管理。不适用于非医疗场景的人员分流。

## Instructions
Follow these steps in order. Ask the user if anything is marked [待确认].

1. 测量生命体征：测量患者体温和血压。
   - 若体温 > 39℃ 或 收缩压 > 180mmHg 或 舒张压 > 110mmHg，标记为「高危」并转至步骤4。
   - 否则，继续步骤2。
2. 询问病史：询问患者主要症状、持续时间、既往病史（如高血压、糖尿病、心脏病等）、过敏史。
   - 若患者主诉胸痛、呼吸困难、意识障碍、大出血，标记为「高危」并转至步骤4。
   - 若患者主诉发热伴咳嗽、腹痛、外伤（无大出血）、头痛，标记为「中危」并转至步骤4。
   - 若患者主诉轻微不适、复诊开药、健康咨询，标记为「低危」并转至步骤4。
   - 若患者无法清晰表达或拒绝回答，标记为「待确认」并转至步骤4。
3. [待确认] 判断分级：根据步骤1和步骤2的结果综合判断患者分级。
   - 若步骤1和步骤2中任一结果为「高危」，最终分级为「高危」。
   - 若步骤1为正常且步骤2为「中危」，最终分级为「中危」。
   - 若步骤1为正常且步骤2为「低危」，最终分级为「低危」。
   - 若步骤1为正常且步骤2为「待确认」，最终分级为「待确认」。
4. 安排科室：根据最终分级和病史安排就诊科室。
   - 若最终分级为「高危」：立即通知急诊科或抢救室，并启动绿色通道。
   - 若最终分级为「中危」且主诉为发热伴咳嗽：安排至呼吸内科。
   - 若最终分级为「中危」且主诉为腹痛：安排至消化内科或普外科 [待确认]。
   - 若最终分级为「中危」且主诉为外伤：安排至骨科或急诊外科 [待确认]。
   - 若最终分级为「中危」且主诉为头痛：安排至神经内科。
   - 若最终分级为「低危」：安排至全科门诊或相应专科门诊 [待确认]。
   - 若最终分级为「待确认」：安排至分诊台由护士进一步评估。

## Decision routes
| 用户意图/条件 | 执行动作 | 备注 |
|------------|---------|------|
| 患者体温 > 39℃ 或血压异常 | 标记高危，转急诊 | 生命体征异常优先处理 |
| 患者主诉胸痛、呼吸困难等 | 标记高危，转急诊 | 典型急症信号 |
| 患者主诉发热咳嗽、腹痛等 | 标记中危，安排对应科室 | 需进一步鉴别 |
| 患者轻微不适或复诊 | 标记低危，安排全科 | 常规处理 |
| 患者无法清晰表达 | 标记待确认，由护士评估 | 信息不足时升级处理 |

## Inputs
- 体温: float，默认值 36.5，范围 35.0-42.0，单位℃
- 收缩压: int，默认值 120，范围 60-250，单位mmHg
- 舒张压: int，默认值 80，范围 30-150，单位mmHg
- 主要症状: string，默认值 ""，可选值 [发热, 咳嗽, 胸痛, 腹痛, 头痛, 外伤, 呼吸困难, 意识障碍, 大出血, 轻微不适, 复诊开药, 健康咨询, 其他]
- 既往病史: string，默认值 "无"，可选值 [无, 高血压, 糖尿病, 心脏病, 其他]
- 过敏史: string，默认值 "无"

## Outputs
- status: enum — 执行结果（success / pending / rejected / escalated）
- message: string — 给用户或下游系统的摘要说明
- audit_notes: string — 关键操作记录（可选）
