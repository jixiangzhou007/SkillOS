# 门诊分诊

## 核心问题
患者挂号后，如何通过标准化流程（一测二问三判断四安排）快速、准确地进行分级并引导至正确科室，同时确保孕妇等特殊人群优先处理。

## S_trigger
- keywords: 分诊, 挂号, 测体温, 测血压, 主诉, 病史, 安排科室, 孕妇, 急诊, 分级
- context: 患者已完成挂号，护士或分诊人员需要执行标准化分诊流程时。
- excludes: 患者未挂号；患者已直接进入抢救室（无需分诊）；用户询问的是药品信息或治疗方案。

## S_body
Follow these steps in order. Ask the user if anything is marked [待确认].

1.  **一测：测量基础生命体征**
    -   测量患者体温、血压。
    -   *if* 体温 > 37.3℃ *or* 血压异常（收缩压 > 140mmHg 或 < 90mmHg，舒张压 > 90mmHg 或 < 60mmHg），标记为“生命体征异常”，进入急诊分诊流程（步骤3）。
    -   *else* 记录正常值，继续步骤2。

2.  **二问：询问主诉及病史**
    -   询问患者主要症状（如：胸痛、呼吸困难、腹痛、外伤等）及持续时间。
    -   询问既往病史（如：心脏病、糖尿病、高血压、过敏史等）。
    -   *if* 患者为孕妇，无论主诉如何，**直接标记为“特殊人群-孕妇”**，跳过分级，进入步骤4（安排科室），并优先安排至妇产科或急诊绿色通道。
    -   *else* 记录主诉和病史，进入步骤3。

3.  **三判断：分级（急诊分诊标准）**
    -   根据生命体征和主诉，按以下标准分级：
        -   **一级（危重）**：生命体征不稳定（如呼吸心跳骤停、严重休克、意识丧失）。→ **立即送入抢救室**，并通知急诊医生。
        -   **二级（危险）**：存在潜在危险（如急性胸痛、严重呼吸困难、大出血）。→ **标记为二级，安排15分钟内接诊**，优先安排急诊医生。
        -   **三级（急症）**：生命体征稳定，但症状明显（如中度腹痛、骨折、高热）。→ **标记为三级，安排候诊**，按顺序排队。
        -   **四级（非急症）**：生命体征稳定，症状轻微（如普通感冒、慢性病复诊、轻微外伤）。→ **标记为四级，安排候诊**，可引导至普通门诊。
    -   *if* 无法明确分级（如主诉模糊、信息不足），标记为“待确认”，转至急诊分诊台由医生复核。

4.  **四安排：安排科室并记录**
    -   根据分级结果和主诉，安排至对应科室：
        -   **一级/二级**：直接安排至**急诊科**。
        -   **三级/四级**：根据主诉安排至**普通门诊**（如：胸痛→心内科；腹痛→消化内科；外伤→外科）。
        -   **孕妇**：无论分级，优先安排至**妇产科**或**急诊绿色通道**。
    -   将所有分诊记录（体温、血压、主诉、病史、分级、安排科室、时间）录入系统。
    -   *if* 当前时间距离上次汇总已满4小时，则执行步骤5。

5.  **数据汇总（每4小时）**
    -   从系统中导出过去4小时内的所有分诊记录。
    -   统计各分级（一级/二级/三级/四级）的患者数量。
    -   统计各科室（急诊科、普通门诊各科室）的接诊量。
    -   生成汇总报告（格式：[待确认] 如：Excel表格或系统内置报表），提交给护士长或值班医生。

```markdown
- 舒张压异常下限: int，默认 60，单位：mmHg，低于此值标记为血压异常
- 二级接诊时限: int，默认 15，单位：分钟，二级患者需在此时间内接诊

- 科室映射规则: dict，默认 {"胸痛": "心内科", "腹痛": "消化内科", "外伤": "外科"}，key为主诉关键词，value为对应科室，支持增删改
- 数据汇总输出格式: enum，默认 "Excel表格"，可选值 [Excel表格, 系统内置报表, CSV文件, JSON接口]
- 汇总报告接收人: list of string，默认 ["护士长", "值班医生"]，可配置多个接收角色或用户ID
```
