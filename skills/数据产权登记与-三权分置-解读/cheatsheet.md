# 速查表: 数据产权登记与“三权分置”解读

```markdown
# 数据产权登记速查表

## 核心公式/规则

> **三权分置框架**
> 数据产权 = 持有权 + 使用权 + 经营权
> （三项权利可分离、可并行授予不同主体）

> **登记凭证 = 信任锚点**
> 统一登记体系 → 权属审查 → 出具权威凭证 → 降低交易成本

> **约定优先、法定补充**
> 数据融合利用时：合同约定 > 法律补充规则

## 决策流程（if-then 分支）

1. **判断数据能否直接流通**
   - 若权属清晰 → 直接交易
   - 若权属模糊 → 先申请数据产权登记凭证

2. **选择授权方式**
   - 若需保障数据安全 → 仅授予使用权或经营权，不授予持有权
   - 若需最大化价值 → 利用“数据要素乘数效应”，将同一数据授权给多个主体并行使用

3. **处理权属争议**
   - 若争议难以厘清 → 从“数据是谁的”转向“数据怎么用”，优先界定使用权

## 常见陷阱

- ❌ **权属不清导致流通不畅**：权利归属模糊 → 交易双方无法建立信任 → 形成“数据孤岛”
- ❌ **忽视数据安全与权益保障**：未建立产权规则和行为红线 → 数据被滥用、泄露或破坏
- ❌ **试图彻底厘清所有权**：在多方共创、低成本复制场景下，应聚焦使用权而非所有权

## 关键数字/阈值

| 指标 | 数值/说明 |
|------|-----------|
| 三权分置 | 持有权、使用权、经营权（三项独立可分离） |
| 授权模式 | 可仅授予1项或2项权利（如仅使用权） |
| 并行授权 | 同一数据可同时授权给多个主体（无上限） |

## 一句话总结

> **数据产权登记制度通过“三权分置”+权威凭证，解决权属不清、流通不畅，是激活数据要素价值的前提基石。**
```
