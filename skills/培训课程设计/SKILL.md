# 培训课程设计

## 核心问题
如何从零开始系统化设计一门培训课程，确保课程内容贴合学员需求、目标明确、效果可评估。

## When to use
- keywords: 培训课程，课程设计，培训方案，课程开发，教学设计，培训流程
- context: 用户需要系统化设计一门培训课程，或梳理课程开发的标准流程
- excludes: 用户仅需要单次培训活动安排（如场地、时间协调），不需要课程内容设计

## Instructions
Follow these steps in order. Ask the user if anything is marked [待确认].

1. **需求调研**：收集学员背景、现有水平、学习目标、业务痛点。调研对象至少覆盖：业务需求方（如经理）、目标学员。若业务需求方目标与学员基线存在明显冲突，需进行差距分析（Gap Analysis），量化期望状态与现状之间的差距。
2. **目标设定**：基于调研结果，明确课程的 SMART 目标（如：学员课后能独立完成XX操作）。目标需与学员基线匹配，不可脱节。
3. **大纲设计**：将目标拆解为模块，设计课程章节、时间分配、逻辑顺序。每节课时长建议不超过45分钟。
4. **课件制作**：制作PPT、讲义、案例、练习题、实操任务。课件需包含互动练习环节。
5. **试讲与反馈**：组织小范围试讲，收集学员/专家反馈，记录修改点。若反馈意见与原始课程目标冲突，应优先回归目标评估反馈的合理性，而非全盘采纳。
6. **正式交付**：按修改后的版本正式授课，记录课堂异常情况。
7. **效果评估**：使用Kirkpatrick四级模型（反应、学习、行为、结果）进行评估。优先使用行为观察和业务数据（如转化率）作为评估依据，而非仅依赖满意度问卷。

## Decision routes
| 用户意图/条件 | 执行动作 | 备注 |
|------------|---------|------|
| 用户需要设计一门新课程 | 执行步骤1-7全流程 | 从需求调研开始 |
| 用户已有调研结果，需要设计大纲和课件 | 从步骤3开始执行 | 跳过调研和目标设定 |
| 用户已完成课程设计，需要评估效果 | 执行步骤7 | 使用Kirkpatrick模型 |
| 试讲反馈与课程目标冲突 | 回归原始调研结论和SMART目标，对反馈进行“支持目标/偏离目标”分类后再决定是否采纳 | 不盲目采纳所有反馈 |
| 学员背景差异过大（经验相差2年以上） | 建议分层教学或分班制，为不同层级设计差异化目标和练习 | 在步骤1或步骤2时判断 |

## Inputs
- 学员信息: object，必填，包含学员背景、现有水平、人数等
- 培训目标: string，必填，业务需求方期望达成的培训效果
- 时长限制: number，选填，课程总时长（分钟），默认值480
- 调研结果: object，选填，已完成的需求调研数据，包含学员基线、业务痛点等
- 课件格式: string，选填，课件输出格式（PPT/PDF/在线文档），默认值PPT
