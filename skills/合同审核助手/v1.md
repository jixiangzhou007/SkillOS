# 技能名称：合同审核助手
## Core Problem
审核采购/销售/服务类合同，识别关键条款变更中的法律与商业风险，并给出修改建议。

| Intent | Action | Resource |
| :--- | :--- | :--- |
| 加载合同 | 上传或粘贴合同文本（Word/PDF/纯文本），系统自动提取全文。 | [待补充：支持的文件格式] |

## S_body
1. **加载合同**：用户上传待审核合同。如果用户提供“基准版本”，则进入比对流程；否则默认与当前生效的标准条款库比对。
2. **条款比对**：自动识别并高亮所有差异。对每处差异，记录“原文”与“改文”。
3. **风险识别**：
   - 若变更涉及**仲裁/诉讼地点**，且从“北京”改为“上海” → 判定为**高风险**（可能增加异地应诉成本、不利管辖倾向）。
   - 若变更涉及**付款周期**（如从30天改为45天） → 判定为**中风险**（现金流影响，可协商）。
   - 若变更涉及**标点/格式** → 判定为**低风险**。
4. **修改建议**：对高风险项，直接给出“必须改回原条款”的建议，并附法律依据（如《民事诉讼法》关于合同履行地管辖的规定）。
5. **导出报告**：按用户需求输出结构化报告。

## 边界情况
- **无基准版本**：提示用户提供，或使用“常见风险点”库进行单版本审查（仅识别明显异常条款，如空白仲裁条款、过高违约金）。
- **多版本混合**：若合同有多处修改且来自不同用户（如甲方改、乙方又改），需标记每处修改的“提出方”。
- **外语合同**：仅支持中英文；其他语言提示“暂不支持，建议人工审核”。

## 用户提示
- 请在上传前确认合同版本为最终版（避免草稿干扰）。
- 如涉及涉外条款，请备注适用法律与语言。
