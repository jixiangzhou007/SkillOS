# 技能名称：合同审核助手
## Core Problem
审核采购/销售/服务类合同，识别关键条款变更中的法律与商业风险，并给出修改建议。
## S_route
| Intent | Action | Resource |
| :--- | :--- | :--- |
| 加载合同 | 上传或粘贴合同文本（Word/PDF/纯文本），系统自动提取全文。 | [待补充：支持的文件格式] |
| 条款比对 | 1. 用户指定**基准版本**（如标准模板/上一版签约合同/谈判记录）。<br>2. 系统逐条比对差异，高亮显示新增、删除、修改内容。<br>3. 输出**变更摘要**（每处变更的位置、原文、改后文）。 | 参考模板库（如市场监管总局示范文本） |
| 风险识别 | 对每处变更，按以下维度评估风险等级：<br>- **高风险**：涉及管辖权、付款条件、违约责任、知识产权归属、保密范围扩大、**核心价格/数量/规格**。<br>- **中风险**：涉及交货/服务时间、验收标准、通知方式、适用法律、**非核心的付款周期**。<br>- **低风险**：涉及格式排版、非关键定义、无实质影响的措辞优化。 | [待补充：风险评级规则库] |

| 导出报告 | 生成含变更对比表、风险等级、修改建议的审核报告（支持PDF/Word/邮件正文）。 | [待补充：导出格式] |

## S_body
1. **加载合同**：用户上传待审核合同。如果用户提供“基准版本”（如上一版合同、谈判记录、报价单），则进入比对流程；否则默认与当前生效的标准条款库比对。
2. **内容与价格初筛**：自动识别合同中的**商品/服务描述、数量、单价、总价、折扣、支付方式**等关键商业条款。若与用户提供的基准版本存在差异，则直接标记为**待确认项**。
3. **条款比对**：自动识别并高亮所有差异。对每处差异，记录“原文”与“改文”。
4. **风险识别**：
   - 若变更涉及**仲裁/诉讼地点**，且从“北京”改为“上海” → 判定为**高风险**（可能增加异地应诉成本、不利管辖倾向）。
   - 若变更涉及**核心价格/数量**（如单价从120元改为100元） → 判定为**高风险**（直接改变合同对价，需立即确认）。
   - 若变更涉及**付款周期**（如从30天改为45天） → 判定为**中风险**（现金流影响，可协商）。
   - 若变更涉及**标点/格式** → 判定为**低风险**。
5. **修改建议**：对高风险项，直接给出“必须改回原条款”的建议，并附法律依据（如《民法典》关于合同对价的规定）；对价格类差异，优先建议“与对方确认是否为笔误”。
6. **导出报告**：生成含变更对比表、风险等级、修改建议的审核报告。
