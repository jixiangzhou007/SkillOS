# 工单处理规范

## 核心问题
标准化 Zendesk 工单处理流程，确保重复工单合并、客户情绪管理、备注格式统一、安全合规上报、周末值班响应及多语言回复的一致性。

## Instructions
Follow these steps in order. Ask the user if anything is marked [待确认].

1.  **判断工单是否重复**：
    -   如果用户明确提到“重复工单”，则执行步骤 2。
    -   否则，跳过合并步骤，直接执行步骤 3。
2.  **合并重复工单**：
    -   将所有重复工单合并到母单（原始工单）。
    -   确保合并后母单保留所有子单的时间线（包括所有备注、更新和事件）。
    -   在母单备注中注明：“已合并重复工单 [子单ID列表]”。
3.  **评估客户情绪与工单优先级**：
    -   **P1/P2 优先级**：如果工单为 P1（生产环境故障）或 P2（核心功能受损），无论客户情绪如何，均直接进入步骤 5（按规范备注与排查）。
    -   **P3/P4 优先级**：如果客户情绪激动（如使用愤怒、焦虑、催促等语气），则执行步骤 4。
    -   **其他情况**：执行步骤 5。
4.  **安抚客户情绪**：
    -   立即回复客户，表达理解和同理心，例如：“感谢您的耐心，我理解您的情况，我们会尽快处理。”
    -   暂缓深入的技术排查，避免在客户情绪激动时进行复杂的技术提问。
    -   在工单备注中注明：“客户情绪激动，已安抚，暂缓技术排查。”
5.  **按规范备注工单**：
    -   无论是否合并或安抚，每次处理工单时，备注必须遵循以下固定格式：
        -   **问题摘要**：用一句话概括客户的核心问题。
        -   **处理步骤**：列出已执行的操作（如：已合并工单、已安抚客户、已排查日志）。
        -   **结论**：当前处理结果或状态（如：等待客户回复、问题已解决、转交研发）。
        -   **待办**：下一步需要执行的操作（如：等待客户提供截图、联系值班工程师）。

## Decision routes
| 用户意图/条件 | 执行动作 | 备注 |
|------------|---------|------|
| 用户明确提到“重复工单” | 执行步骤2：合并重复工单 | 合并到母单，保留时间线，备注子单ID列表 |
| 客户情绪激动（如愤怒、催促）且工单优先级为P3/P4 | 执行步骤4：安抚客户情绪 | 暂缓技术排查 |
| 工单优先级为P1/P2 | 跳过情绪安抚，直接执行步骤5 | 故障优先，快速响应 |
| 多语言工单 | 使用客户使用的语言回复 | 确保沟通顺畅 |
| 涉及数据安全或合规问题 | 走专门通道，禁止口头承诺修复时间 | 记录并上报安全/合规团队 |
| 周末值班且工单为P1 | 电话联系客户 | 快速响应生产故障 |
| 需与研发协作 | 在Jira创建bug，关联原Zendesk工单 | 确保问题可追踪 |
