# 技能名称：测试抽取流程（Test Extraction Flow）- 草稿 v0.2

## 元数据
- **类型**: MetaSkill 流水线
- **版本**: 0.2
- **触发条件**: 用户提交订单时触发
- **核心目标**: 从用户提交的订单数据中，按照既定规则准确抽取目标信息，并处理边界情况。

## 流水线步骤

### 阶段一：前置条件与输入定义
- **输入**:
  - 用户提交的完整订单数据（结构化为JSON或表单）。
  - [已确认] 包含订单ID和客户ID。
  - [待补充] 是否包含文件（如合同、凭证）？需要用户确认。
- **前置条件**:
  - 订单数据已通过基础校验（如必填字段不为空）。
  - [待补充] 抽取规则（如字段优先级、冲突处理策略）是否已预定义？

### 阶段二：执行步骤
1.  **规则匹配**: 根据预定义的抽取规则，扫描订单数据中的目标字段。
2.  **冲突检测**: 检查是否存在名称相似或语义重叠的字段（例如“商品A（主商品）”与“商品A（赠品）”）。
    - **边界情况处理**:
      - **若存在冲突**:
        - 第一步：判断规则库中是否有针对此类冲突的优先级定义。
          - 若已定义（例如“主商品优先于赠品”），则按规则处理，并记录处理日志。
          - 若未定义，则**暂停流程，输出冲突字段详情，并报错等待人工介入**。
        - [新增] 风险等级：字段冲突且无规则定义时，风险等级为 **risk_high**。
      - 若规则未定义，则报错并暂停流程，等待人工定义规则。
3.  **数据抽取**: 从订单数据中提取匹配到的目标字段值。
4.  **格式转换（可选）**: [待补充] 根据下游系统要求，将抽取结果转换为指定格式（如纯文本、JSON）。

### 阶段三：输出与产出
- **标准输出**: 一个包含已抽取字段的键值对集合（如 `{“商品A（主商品）”: “xxx”}`）。
- **异常输出**: 若流程中断，输出错误日志和冲突字段详情，供人工排查。
- [待补充] 输出是否需包含抽取过程的置信度或日志？

### 阶段四：边界情况与异常处理
- **字段缺失**: 若目标字段不存在，则输出空值并记录日志，不中断流程。
- **字段冲突**: 如上所述，依据规则处理或报错。
- **数据格式异常**: 例如字段值应为数字但包含文字，则尝试清洗转换，失败则报错。
- [待补充] 订单中含有多条相同记录时，如何抽取（取第一条/最后一条/合并）？
