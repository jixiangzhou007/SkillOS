### 技能文档：售后订单处理与退款审核

**技能描述**：负责处理用户提交的售后申请，包括退款、退货、换货请求，并根据平台规则与内部流程进行审核、流转与最终处理。

**核心职责**：
1.  **接收与分类售后申请**：从客服系统接收用户提交的售后工单，根据问题类型（退款、退货、换货）与订单状态进行分类。
2.  **初步审核与信息校验**：
    -   核对用户身份与订单信息是否一致。
    -   检查申请是否在售后时效内（如：签收后7天内）。
    -   确认商品状态是否符合退货/换货条件（如：不影响二次销售）。
    -   虚拟商品发货后不支持退款：若订单为虚拟商品（如充值卡、电子券）且已发货，直接驳回退款申请，并告知用户“虚拟商品发货后不支持退款”。
3.  **金额与权限判断**：
    -   若退款金额 **≤ 500元**，由客服主管直接审批。
    -   **若退款金额 > 500元，需转交至业务主管审批，并通过飞书发送审批留痕消息（包含工单编号、金额、原因），待主管确认后方可执行。审批完成后，需将飞书审批截图保存至工单附件，作为权限留痕凭证。**
4.  **物流与签收状态处理**：
    -   已发货但未签收：优先尝试联系快递拦截。若拦截成功，按退款流程处理；若拦截失败，需转交主管审批，由主管决定是否先行退款或等待用户签收后处理退货。
    -   已签收：引导用户退回商品，并提供退货地址。
5.  **换货处理**：
    -   换货申请需生成独立换货工单，与原订单解耦，并按照换货流程单独跟进发货与库存。
    -   确认库存充足后，通知用户寄回旧商品；收到旧商品并质检通过后，发出
6.  **特殊场景处理**：
    -   **买家威胁差评升级主管但不默认全额退款**：若用户以“给差评”为由要求退款，不得直接同意全额退款。**需将工单升级至主管处理，由主管评估问题性质后决定解决方案（如部分退款、补偿优惠券或坚持原判），严禁客服自行承诺全额退款以换取好评。**
