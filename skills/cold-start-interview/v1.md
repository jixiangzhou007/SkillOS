# 技能名称：cold-start-interview

## S_body

冷启动访谈的目标：在创建任何 skill 之前，先花 10-15 分钟把团队的"家法"收进一个共享上下文文件。后续所有 skill 萃取和优化都从这里读你的 playbook、模板、风格偏好。

### 访谈六阶段

**阶段 1: 团队画像 (Team Identity)**
- 问：你们的团队是做什么的？属于哪个行业？
- 问：团队的规模和角色分布？
- 问：日常最重复、最想自动化的 3 件事是什么？
- 产出：一段 100 字的团队画像

**阶段 2: 文档标准 (Document Standards)**
- 问：你们常用的文档类型有哪些？（合同、备忘录、报告、邮件……）
- 问：有没有格式模板或范例可以参照？
- 问：对输出格式有什么硬性要求？（如：必须用 Word track changes、必须 Markdown、必须中文）
- 产出：文档标准清单

**阶段 3: 风格偏好 (Style Preferences)**
- 问：输出风格偏好？正式/半正式/口语？啰嗦/简洁？
- 问：有没有绝对不能用的词或表述？（如：不能出现"建议咨询律师"、"仅供参考"）
- 问：参考哪个团队成员的写作风格最接近你们想要的？
- 产出：风格指南

**阶段 4: 术语表 (Term Glossary)**
- 问：你们行业/团队内部有哪些特殊术语、缩写、黑话？
- 问：哪些词有特定的含义（和通用含义不同）？
- 问：中英文混用的规则是什么？
- 产出：术语表

**阶段 5: 工作流与习惯 (Workflow & Playbook)**
- 问：你们的审批流程是什么？（谁起草 → 谁审核 → 谁签发）
- 问：合同/文档的命名规范？
- 问：文件存在哪里？（本地/网盘/DMS/邮件附件）
- 产出：工作流说明

**阶段 6: 工具与系统 (Tools & Systems)**
- 问：你们日常用什么工具？（Word/飞书/Notion/Slack/邮件……）
- 问：有没有现有的 MCP 连接器或 API 可以用？
- 产出：工具清单

### 输出

访谈结束后，生成一个 `PLAYBOOK.md` 文件，存放在 `skills/.claude/PLAYBOOK.md`。

此文件被所有后续 skill 萃取和优化流程自动读取。Skill 生成时会加入"对照 playbook 检查"步骤。

## S_trigger

- **keywords**: cold start, 冷启动, 初始化, 首次使用, 新手引导, setup, onboarding, 团队配置, 我的团队, 我们公司, playbook, 家法
- **context**: 新用户首次使用、新团队/新项目初始化、用户想建立团队知识基线
- **excludes**: 创建技能, 萃取, 对话萃取（这些是 skill-creator 的范畴）

## S_params

- model: deepseek-v4-flash
- output_path: skills/.claude/PLAYBOOK.md
- interview_time_estimate: 10-15 minutes

## S_appendix

### PLAYBOOK.md 格式

```markdown
# Team Playbook

## 团队画像
[一段话描述团队是做什么的]

## 文档标准
- 常用文档类型: [...]
- 格式模板: [...]
- 硬性要求: [...]

## 风格偏好
- 正式度: [...]
- 简洁度: [...]
- 禁用词: [...]
- 参考风格: [...]

## 术语表
| 术语 | 定义 | 备注 |
|------|------|------|
| ... | ... | ... |

## 工作流
- 起草人: [...]
- 审核人: [...]
- 签发人: [...]
- 命名规范: [...]

## 工具与系统
- [...]
```

### 与 Skill 萃取的集成

创建 skill 时，S_body 最后一步应加入：
```
对照 PLAYBOOK.md 检查：
1. 输出格式是否符合文档标准？
2. 用语是否匹配风格偏好？
3. 术语是否使用了团队定义？
4. 是否集成了团队已有的工具？
```
