# 低于需改进线的员工进入P

## 核心问题
将绩效评估低于需改进线的员工自动纳入PIP（绩效改进计划）流程，并设置60天观察期进行跟踪管理。

## When to use
- keywords: PIP, 绩效改进计划, 需改进线, 观察期, 60天, 绩效不达标, 低于标准
- context: 员工绩效评估结果出来后，需要将低于需改进线的员工纳入改进计划
- excludes: 员工绩效高于需改进线、员工主动离职、员工已处于其他改进计划中

## Instructions
Follow these steps in order. Ask the user if anything is marked [待确认].

1. 确认员工当前绩效评估结果是否低于需改进线
   - 如果用户提供了绩效分数或评级，判断是否低于需改进线阈值
   - 如果用户未提供，询问：「该员工的绩效评估结果是什么？是否低于需改进线？」
2. 确认员工进入PIP流程
   - 告知用户该员工符合PIP条件
   - 确认用户是否要启动PIP流程
3. 设置60天观察期
   - 从当前日期开始计算，设置60天观察期
   - 记录观察期开始日期和结束日期
   - 如果用户指定了其他观察期时长，[待确认]是否覆盖默认60天
4. 生成PIP计划文档
   - 包含员工基本信息
   - 包含需改进的具体领域
   - 包含60天观察期起止时间
   - 包含改进目标和里程碑
   - [待确认]是否需要包含具体改进措施模板
5. 通知相关方
   - [待确认]需要通知哪些人员（HR、直属上级、员工本人等）
   - [待确认]通知方式（邮件、系统通知等）

## Decision routes
| 用户意图/条件 | 执行动作 | 备注 |
|------------|---------|------|
| 用户提供员工绩效低于需改进线 | 直接进入PIP流程，设置60天观察期 | 默认流程 |
| 用户询问PIP流程如何操作 | 解释PIP流程步骤，确认是否要启动 | 引导式流程 |
| 用户要求修改观察期时长 | [待确认]是否允许修改，需确认规则 | 需确认 |
| 用户要求取消PIP流程 | 取消当前操作，记录取消原因 | 异常处理 |

## Inputs
- employee_name: string, 必填, 员工姓名
- employee_id: string, 可选, 员工工号
- performance_result: string, 必填, 绩效评估结果（分数或评级）
- observation_days: integer, 默认60, 观察期天数
- improvement_areas: array, 可选, 需改进的具体领域列表
- start_date: date, 默认当天, 观察期开始日期

## Outputs
- status: enum — 执行结果（success / pending / rejected / escalated）
- message: string — 给用户或下游系统的摘要说明
- audit_notes: string — 关键操作记录（可选）
