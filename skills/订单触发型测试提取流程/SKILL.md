# 技能名称：订单触发型测试提取流程（Order-Triggered Test Extraction Flow）
## 能力
- 根据用户订单提交事件，自动启动测试提取流程。
- 支持在流程开始时对订单参数进行合法性校验（如金额、商品ID、用户状态）。
- 支持根据订单的核心特征（如促销类型、金额边界）动态选择优先提取的测试用例组。
- [待补充] 支持提取结果的优先级排序与冲突处理规则。
- [待补充] 支持提取失败时的回退与日志记录机制。

## 约束
- 提取流程必须在订单提交后5秒内触发，避免影响用户体验。
- 仅提取与当前订单直接相关的测试用例，不进行全量用例扫描。
- [待补充] 若订单参数不合法，应中止提取并返回错误码，不执行任何测试。
- [待补充] 对于金额为0、商品ID无效等边界或非法情况，需定义明确的分支处理逻辑（例如：中止、警告后继续、或视为特殊场景继续）。
- [新增] 对于被标记为“高风险客户”的订单，必须在验证阶段决定是中止流程还是进入增强审核分支。

## 流程（草案）
1. **触发**：监听用户提交订单事件，捕获订单ID、金额、商品ID、用户ID等关键字段。
2. **校验**：对金额、商品ID等参数进行基础合法性检查（如金额>0、商品ID存在）。
   - [待补充] 若金额为0，当前草案未定义处理分支，需根据业务规则确定是“中止”还是“标记为边界场景后继续”。
   - [新增] **客户风险校验**：根据用户ID查询客户风险等级（如低风险、高风险）。若为高风险，则根据配置决定是“中止并报警”还是“标记后继续进入后续步骤”。
3. **提取策略选择**：根据订单特征（如促销类型、金额边界、用户等级）选择优先提取的测试用例组。
   - 示例：若订单为“满减”且金额等于临界值，优先提取“边界值测试用例”。
   - [待补充] 当多个策略同时匹配时，需定义冲突处理规则（如按优先级顺序执行、合并去重、或暂停等待人工裁决）。
4. **执行提取**：调用测试用例库API，根据策略返回匹配的用例ID列表。
5. **记录**：将提取结果（用例ID、提取时间、触发订单ID）写入日志。
